\doxysubsection{Detailed Description}
Interface used for database preparation where we first have to register all data types (and memory output locations) before extracting data, e.\+g. O\+D\+BC. Extract works as two-\/phase extract\+: first we call prepare once, then extract n-\/times. There are cases (bulk operations using std\+::vector storage) when extract is called only once. The value passed to a \mbox{\hyperlink{classPoco_1_1Data_1_1AbstractPreparator_acfa28a168ed828b272313d3d380af3f5}{prepare()}} call is not used by the prepare, serving only as an indication of the data type being prepared, thus all values are passed as const references. Implementing this interface is not mandatory for a connector. Connectors that only extract data after S\+QL execution (e.\+g. \mbox{\hyperlink{namespacePoco_1_1Data_1_1SQLite}{S\+Q\+Lite}}) do not need this functionality at all.
