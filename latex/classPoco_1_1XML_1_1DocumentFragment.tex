\doxysubsection{Detailed Description}
\mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}} is a \char`\"{}lightweight\char`\"{} or \char`\"{}minimal\char`\"{} \mbox{\hyperlink{classPoco_1_1XML_1_1Document}{Document}} object. It is very common to want to be able to extract a portion of a document\textquotesingle{}s tree or to create a new fragment of a document. Imagine implementing a user command like cut or rearranging a document by moving fragments around. It is desirable to have an object which can hold such fragments and it is quite natural to use a \mbox{\hyperlink{classPoco_1_1XML_1_1Node}{Node}} for this purpose. While it is true that a \mbox{\hyperlink{classPoco_1_1XML_1_1Document}{Document}} object could fulfill this role, a \mbox{\hyperlink{classPoco_1_1XML_1_1Document}{Document}} object can potentially be a heavyweight object, depending on the underlying implementation. What is really needed for this is a very lightweight object. \mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}} is such an object.

Furthermore, various operations -- such as inserting nodes as children of another \mbox{\hyperlink{classPoco_1_1XML_1_1Node}{Node}} -- may take \mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}} objects as arguments; this results in all the child nodes of the \mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}} being moved to the child list of this node.

The children of a \mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}} node are zero or more nodes representing the tops of any sub-\/trees defining the structure of the document. \mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}} nodes do not need to be well-\/formed \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} documents (although they do need to follow the rules imposed upon well-\/formed \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} parsed entities, which can have multiple top nodes). For example, a \mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}} might have only one child and that child node could be a \mbox{\hyperlink{classPoco_1_1XML_1_1Text}{Text}} node. Such a structure model represents neither an H\+T\+ML document nor a well-\/formed \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} document.

When a \mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}} is inserted into a \mbox{\hyperlink{classPoco_1_1XML_1_1Document}{Document}} (or indeed any other \mbox{\hyperlink{classPoco_1_1XML_1_1Node}{Node}} that may take children) the children of the \mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}} and not the \mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}} itself are inserted into the \mbox{\hyperlink{classPoco_1_1XML_1_1Node}{Node}}. This makes the \mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}} very useful when the user wishes to create nodes that are siblings; the \mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}} acts as the parent of these nodes so that the user can use the standard methods from the \mbox{\hyperlink{classPoco_1_1XML_1_1Node}{Node}} interface, such as insert\+Before and append\+Child.
