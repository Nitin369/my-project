\doxysubsection{Detailed Description}
This class provides a simple interface for accessing a serial port.

There are two modes of operation for receiving data\+: polling (blocking) mode (default) and events (callback) mode.

In polling mode, repeatedly call \mbox{\hyperlink{classIoT_1_1Devices_1_1SerialDeviceRemoteObject_a3c4274006697a8f686fe21ef5057a923}{poll()}} to detect if data is available, then call one of the read() methods to receive available data. The read() methods can also be called if no data is available, but will block until at least one character can be read.

In event mode the class will start a thread that polls and reads available data from the serial port, and fires the line\+Received event whenever something has been read. Since callback mode uses \mbox{\hyperlink{classIoT_1_1Devices_1_1SerialDeviceRemoteObject_ae4cd3f082f9f2660a8c23bb7883fd8f4}{read\+Line()}} to receive entire lines (if possible), delimiter characters and timeout can be set via the \char`\"{}delimiters\char`\"{} (string) and \char`\"{}timeout\char`\"{} (double) properties.

Clients of the class can switch between polling and event mode by enabling or disabling the \char`\"{}events\char`\"{} feature\+: \begin{DoxyVerb}setFeature("events", true); 
\end{DoxyVerb}
