\doxysubsection{Detailed Description}
Default base class for S\+A\+X2 event handlers. This class is available as a convenience base class for S\+A\+X2 applications\+: it provides default implementations for all of the callbacks in the four core S\+A\+X2 handler classes\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{classPoco_1_1XML_1_1EntityResolver}{Entity\+Resolver}}
\item \mbox{\hyperlink{classPoco_1_1XML_1_1DTDHandler}{D\+T\+D\+Handler}}
\item \mbox{\hyperlink{classPoco_1_1XML_1_1ContentHandler}{Content\+Handler}}
\item \mbox{\hyperlink{classPoco_1_1XML_1_1ErrorHandler}{Error\+Handler}} Application writers can extend this class when they need to implement only part of an interface; parser writers can instantiate this class to provide default handlers when the application has not supplied its own. 
\end{DoxyItemize}

Receive notification of the beginning of an element.

The Parser will invoke this method at the beginning of every element in the \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} document; there will be a corresponding end\+Element event for every start\+Element event (even when the element is empty). All of the element\textquotesingle{}s content will be reported, in order, before the corresponding end\+Element event.

This event allows up to three name components for each element\+:
\begin{DoxyEnumerate}
\item the Namespace \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}};
\item the local name; and
\item the qualified (prefixed) name.
\end{DoxyEnumerate}

\mbox{\hyperlink{classPoco_1_1Any}{Any}} or all of these may be provided, depending on the values of the \href{http://xml.org/sax/features/namespaces}{\texttt{ http\+://xml.\+org/sax/features/namespaces}} and the \href{http://xml.org/sax/features/namespace-prefixes}{\texttt{ http\+://xml.\+org/sax/features/namespace-\/prefixes}} properties\+:
\begin{DoxyItemize}
\item the Namespace \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}} and local name are required when the namespaces property is true (the default), and are optional when the namespaces property is false (if one is specified, both must be);
\item the qualified name is required when the namespace-\/prefixes property is true, and is optional when the namespace-\/prefixes property is false (the default).
\end{DoxyItemize}

Note that the attribute list provided will contain only attributes with explicit values (specified or defaulted)\+: \#\+I\+M\+P\+L\+I\+ED attributes will be omitted. The attribute list will contain attributes used for Namespace declarations (xmlns$\ast$ attributes) only if the \href{http://xml.org/sax/features/namespace-prefixes}{\texttt{ http\+://xml.\+org/sax/features/namespace-\/prefixes}} property is true (it is false by default, and support for a true value is optional).

Like \mbox{\hyperlink{classPoco_1_1XML_1_1DefaultHandler_accc51506e28a6076d25b64cf95b34d80}{characters()}}, attribute values may have characters that need more than one char value.

Begin the scope of a prefix-\/\+U\+RI Namespace mapping.

The information from this event is not necessary for normal Namespace processing\+: the S\+AX \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} reader will automatically replace prefixes for element and attribute names when the \href{http://xml.org/sax/features/namespaces}{\texttt{ http\+://xml.\+org/sax/features/namespaces}} feature is true (the default).

There are cases, however, when applications need to use prefixes in character data or in attribute values, where they cannot safely be expanded automatically; the start/end\+Prefix\+Mapping event supplies the information to the application to expand prefixes in those contexts itself, if necessary.

Note that start/end\+Prefix\+Mapping events are not guaranteed to be properly nested relative to each other\+: all start\+Prefix\+Mapping events will occur immediately before the corresponding start\+Element event, and all end\+Prefix\+Mapping events will occur immediately after the corresponding end\+Element event, but their order is not otherwise guaranteed.

There should never be start/end\+Prefix\+Mapping events for the \char`\"{}xml\char`\"{} prefix, since it is predeclared and immutable.

This corresponds to the definition of \char`\"{}error\char`\"{} in section 1.\+2 of the W3C \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} 1.\+0 Recommendation. For example, a validating parser would use this callback to report the violation of a validity constraint. The default behaviour is to take no action.

If the application cannot do so, then the parser should report a fatal error even if the \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} recommendation does not require it to do so.

Filters may use this method to report other, non-\/\+X\+ML errors as well.

Receive notification of character data.

The Parser will call this method to report each chunk of character data. S\+AX parsers may return all contiguous character data in a single chunk, or they may split it into several chunks; however, all of the characters in any single event must come from the same external entity so that the \mbox{\hyperlink{classPoco_1_1XML_1_1Locator}{Locator}} provides useful information.

The application must not attempt to read from the array outside of the specified range.

Individual characters may consist of more than one X\+M\+L\+Char value. There are three important cases where this happens, because characters can\textquotesingle{}t be represented in just sixteen bits. In one case, characters are represented in a Surrogate Pair, using two special \mbox{\hyperlink{classPoco_1_1Unicode}{Unicode}} values. Such characters are in the so-\/called \char`\"{}\+Astral Planes\char`\"{}, with a code point above U+\+F\+F\+FF. A second case involves composite characters, such as a base character combining with one or more accent characters. And most important, if X\+M\+L\+Char is a plain char, characters are encoded in U\+T\+F-\/8.

Your code should not assume that algorithms using char-\/at-\/a-\/time idioms will be working in character units; in some cases they will split characters. This is relevant wherever \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} permits arbitrary characters, such as attribute values, processing instruction data, and comments as well as in data reported from this method. It\textquotesingle{}s also generally relevant whenever C++ code manipulates internationalized text; the issue isn\textquotesingle{}t unique to \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}}.

Note that some parsers will report whitespace in element content using the ignorable\+Whitespace method rather than this one (validating parsers must do so).

The parser will call this method before opening any external entity except the top-\/level document entity. Such entities include the external D\+TD subset and external parameter entities referenced within the D\+TD (in either case, only if the parser reads external parameter entities), and external general entities referenced within the document element (if the parser reads external general entities).
