\doxysubsection{Detailed Description}
This class implements a Nested Diagnostic Context (N\+DC), as described in Neil Harrison\textquotesingle{}s article \char`\"{}\+Patterns for Logging 
    Diagnostic Messages\char`\"{} in \char`\"{}\+Pattern Languages of Program Design 3\char`\"{} (Addison-\/\+Wesley).

A N\+DC maintains a stack of context information, consisting of an informational string (e.\+g., a method name), as well as an optional source code line number and file name. N\+D\+Cs are especially useful for tagging log messages with context information which is very helpful in a multithreaded server scenario. Every thread has its own private N\+DC, which is automatically created when needed and destroyed when the thread terminates.

The \mbox{\hyperlink{classPoco_1_1NDCScope}{N\+D\+C\+Scope}} (or \mbox{\hyperlink{classPoco_1_1NestedDiagnosticContext_a449bd8481908e7a72ba259fb9970bdea}{N\+D\+C\+::\+Scope}}) class can be used to automatically push information at the beginning of a scope, and to pop it at the end. The \mbox{\hyperlink{NestedDiagnosticContext_8h_a93e209c8d31ca16b50da3bd859cc286a}{poco\+\_\+ndc(info)}} macro augments the information with a source code line number and file name.
