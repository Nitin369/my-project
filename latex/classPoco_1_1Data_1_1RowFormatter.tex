\doxysubsection{Detailed Description}
\mbox{\hyperlink{classPoco_1_1Data_1_1Row}{Row}} formatter is an abstract class providing definition for row formatting functionality. For custom formatting strategies, inherit from this class and override \mbox{\hyperlink{classPoco_1_1Data_1_1RowFormatter_ac26e6aa4b71bfb277e9e44790831d6d3}{format\+Names()}} and \mbox{\hyperlink{classPoco_1_1Data_1_1RowFormatter_a999ace0f3b39f48a603ff388523ef88e}{format\+Values()}} member functions.

\mbox{\hyperlink{classPoco_1_1Data_1_1Row}{Row}} formatter can be either passed to the \mbox{\hyperlink{classPoco_1_1Data_1_1RecordSet}{Record\+Set}} at construction time, like in the following example\+:

\mbox{\hyperlink{classPoco_1_1Data_1_1RecordSet}{Record\+Set}} rs(session. \char`\"{}\+S\+E\+L\+E\+C\+T $\ast$ F\+R\+O\+M Table\char`\"{}, new My\+Row\+Formater);

or it can be supplied to the statement as in the following example\+:

My\+Row\+Formatter rf session $<$$<$ \char`\"{}\+S\+E\+L\+E\+C\+T $\ast$ F\+R\+O\+M Table\char`\"{}, format(rf);

If no formatter is externally supplied to the statement, the \mbox{\hyperlink{classPoco_1_1Data_1_1SimpleRowFormatter}{Simple\+Row\+Formatter}} is used. \mbox{\hyperlink{classPoco_1_1Data_1_1Statement}{Statement}} always has the ownership of the row formatter and shares it with rows through \mbox{\hyperlink{classPoco_1_1Data_1_1RecordSet}{Record\+Set}}.

To accomodate for various formatting needs, a formatter can operate in two modes\+:


\begin{DoxyItemize}
\item progressive\+: formatted individual row strings are gemerated and returned from each call to format\+Values; std\+::string\& \mbox{\hyperlink{classPoco_1_1Data_1_1RowFormatter_ac26e6aa4b71bfb277e9e44790831d6d3}{format\+Names(const Name\+Vec\+Ptr, std\+::string\&)}} and std\+::string\& \mbox{\hyperlink{classPoco_1_1Data_1_1RowFormatter_a999ace0f3b39f48a603ff388523ef88e}{format\+Values(const Value\+Vec\&, std\+::string\&)}} member calls should be used in this case; this is the default mode
\item bulk\+: formatted resulting string is accumulated internally and obtained at the end of iteration via \mbox{\hyperlink{classPoco_1_1Data_1_1RowFormatter_aa551bb4c9de06f9e88718600235eab2c}{to\+String()}} member function; void \mbox{\hyperlink{classPoco_1_1Data_1_1RowFormatter_a199767ea3ff58be71298a0b10636ea26}{format\+Names(const Name\+Vec\+Ptr)}} and void \mbox{\hyperlink{classPoco_1_1Data_1_1RowFormatter_a779ffbb6dd82c572cf0ead3e633c0d05}{format\+Values(const Value\+Vec\&)}} member calls should be used in this case
\end{DoxyItemize}

When formatter is used in conjunction with Row/\+Record\+Set, the formatting members corresponding to the formater mode are expected to be implemented. If a call is propagated to this parent class, the functions do nothing or silently return empty string respectively.
