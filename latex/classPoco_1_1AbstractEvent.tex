\doxysubsection{Detailed Description}
\subsubsection*{template$<$class T\+Args, class T\+Strategy, class T\+Delegate, class T\+Mutex = Fast\+Mutex$>$\newline
class Poco\+::\+Abstract\+Event$<$ T\+Args, T\+Strategy, T\+Delegate, T\+Mutex $>$}

An \mbox{\hyperlink{classPoco_1_1AbstractEvent}{Abstract\+Event}} is the base class of all events. It works similar to the way C\# handles notifications (aka events in C\#).

Events can be used to send information to a set of delegates which are registered with the event. The type of the data is specified with the template parameter T\+Args. The T\+Strategy parameter must be a subclass of \mbox{\hyperlink{classPoco_1_1NotificationStrategy}{Notification\+Strategy}}. The parameter T\+Delegate can either be a subclass of \mbox{\hyperlink{classPoco_1_1AbstractDelegate}{Abstract\+Delegate}} or of \mbox{\hyperlink{classPoco_1_1AbstractPriorityDelegate}{Abstract\+Priority\+Delegate}}.

Note that \mbox{\hyperlink{classPoco_1_1AbstractEvent}{Abstract\+Event}} should never be used directly. One ought to use one of its subclasses which set the T\+Strategy and T\+Delegate template parameters to fixed values. For most use-\/cases the \mbox{\hyperlink{classPoco_1_1BasicEvent}{Basic\+Event}} template will be sufficient\+: \begin{DoxyVerb}#include "Poco/BasicEvent.h"
#include "Poco/Delegate.h"
\end{DoxyVerb}


Note that as of release 1.\+4.\+2, the behavior of \mbox{\hyperlink{classPoco_1_1BasicEvent}{Basic\+Event}} equals that of \mbox{\hyperlink{classPoco_1_1FIFOEvent}{F\+I\+F\+O\+Event}}, so the \mbox{\hyperlink{classPoco_1_1FIFOEvent}{F\+I\+F\+O\+Event}} class is no longer necessary and provided for backwards compatibility only.

\mbox{\hyperlink{classPoco_1_1BasicEvent}{Basic\+Event}} works with a standard delegate. They allow one object to register one or more delegates with an event. In contrast, a \mbox{\hyperlink{classPoco_1_1PriorityDelegate}{Priority\+Delegate}} comes with an attached priority value and allows one object to register for one priority value one or more delegates. Note that Priority\+Delegates only work with Priority\+Events\+: \begin{DoxyVerb}#include "Poco/PriorityEvent.h"
#include "Poco/PriorityDelegate.h"
\end{DoxyVerb}


Use events by adding them as public members to the object which is throwing notifications\+: \begin{DoxyVerb}class MyData
{
public:
    Poco::BasicEvent<int> dataChanged;

    MyData();
    ...
    void setData(int i);
    ...
private:
    int _data;
};
\end{DoxyVerb}


Firing the event is done either by calling the event\textquotesingle{}s \mbox{\hyperlink{classPoco_1_1AbstractEvent_a13197de5929f8db8440cab9b2ca4d713}{notify()}} or \mbox{\hyperlink{classPoco_1_1AbstractEvent_aed56afbf8d9a88baf16595ef23b9fca0}{notify\+Async()}} method\+: \begin{DoxyVerb}void MyData::setData(int i)
{
    this->_data = i;
    dataChanged.notify(this, this->_data);
}
\end{DoxyVerb}


Alternatively, instead of \mbox{\hyperlink{classPoco_1_1AbstractEvent_a13197de5929f8db8440cab9b2ca4d713}{notify()}}, operator () can be used. \begin{DoxyVerb}void MyData::setData(int i)
{
    this->_data = i;
    dataChanged(this, this->_data);
}
\end{DoxyVerb}


Note that operator (), \mbox{\hyperlink{classPoco_1_1AbstractEvent_a13197de5929f8db8440cab9b2ca4d713}{notify()}} and \mbox{\hyperlink{classPoco_1_1AbstractEvent_aed56afbf8d9a88baf16595ef23b9fca0}{notify\+Async()}} do not catch exceptions, i.\+e. in case a ~\newline
 delegate throws an exception, notifying is immediately aborted and the exception is propagated back to the caller.

Delegates can register methods at the event. In the case of a \mbox{\hyperlink{classPoco_1_1BasicEvent}{Basic\+Event}} the \mbox{\hyperlink{classPoco_1_1Delegate}{Delegate}} template is used, in case of an \mbox{\hyperlink{classPoco_1_1PriorityEvent}{Priority\+Event}} a \mbox{\hyperlink{classPoco_1_1PriorityDelegate}{Priority\+Delegate}} is used. Mixing of delegates, e.\+g. using a \mbox{\hyperlink{classPoco_1_1PriorityDelegate}{Priority\+Delegate}} with a \mbox{\hyperlink{classPoco_1_1BasicEvent}{Basic\+Event}} is not allowed and can lead to compile-\/time and/or run-\/time errors. The standalone \mbox{\hyperlink{namespacePoco_a3c92b935465d543414d807c7884396ff}{delegate()}} functions can be used to construct \mbox{\hyperlink{classPoco_1_1Delegate}{Delegate}} objects.

Events require the observers to have one of the following method signatures\+: \begin{DoxyVerb}void onEvent(const void* pSender, TArgs& args);
void onEvent(TArgs& args);
static void onEvent(const void* pSender, TArgs& args);
static void onEvent(void* pSender, TArgs& args);
static void onEvent(TArgs& args);
\end{DoxyVerb}


For performance reasons arguments are always sent by reference. This also allows observers to modify the event argument. To prevent that, use $<$\mbox{[}const T\+Arg\mbox{]}$>$ as template parameter. A non-\/conformant method signature leads to compile errors.

Assuming that the observer meets the method signature requirement, it can register this method with the += operator\+: \begin{DoxyVerb}class MyController
{
protected:
    MyData _data;

    void onDataChanged(void* pSender, int& data);
    ...
};

MyController::MyController()
{
    _data.dataChanged += delegate(this, &MyController::onDataChanged);
}
\end{DoxyVerb}


In some cases it might be desirable to work with automatically expiring registrations. Simply add to delegate as 3rd parameter a expire\+Value (in milliseconds)\+: \begin{DoxyVerb}_data.dataChanged += delegate(this, &MyController::onDataChanged, 1000);
\end{DoxyVerb}


This will add a delegate to the event which will automatically be removed in 1000 millisecs.

Unregistering happens via the -\/= operator. Forgetting to unregister a method will lead to segmentation faults later, when one tries to send a notify to a no longer existing object. \begin{DoxyVerb}MyController::~MyController()
{
    _data.dataChanged -= delegate(this, &MyController::onDataChanged);
}
\end{DoxyVerb}


Working with \mbox{\hyperlink{classPoco_1_1PriorityDelegate}{Priority\+Delegate}}\textquotesingle{}s as similar to working with \mbox{\hyperlink{classPoco_1_1BasicEvent}{Basic\+Event}}. Instead of \mbox{\hyperlink{namespacePoco_a3c92b935465d543414d807c7884396ff}{delegate()}}, the \mbox{\hyperlink{namespacePoco_a2f48d7b2c7774e22e565fc0afb97218f}{priority\+Delegate()}} function must be used to create the \mbox{\hyperlink{classPoco_1_1PriorityDelegate}{Priority\+Delegate}}.
