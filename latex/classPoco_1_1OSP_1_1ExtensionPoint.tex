\doxysubsection{Detailed Description}
An extension point allows a bundle to provide \char`\"{}hooks\char`\"{} that other bundles can hook into to extend a bundle\textquotesingle{}s functionality.

A bundle registers one or more instances of subclasses of \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPoint}{Extension\+Point}} with the \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPointService}{Extension\+Point\+Service}}. The \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPointService}{Extension\+Point\+Service}} then looks for an \char`\"{}extensions.\+xml\char`\"{} file in every bundle that is started and calls the \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPoint_a667347045bd8acccd6fac30092c33e1a}{handle\+Extension()}} member function for every extension element targetted at the extension point.

When a bundle defining an extension is stopped, the \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPoint}{Extension\+Point}}\textquotesingle{}s \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPoint_abb4b377d97404fa6e833d6282276a9f4}{remove\+Extension()}} member function is invoked.

Handles the \char`\"{}extension\char`\"{} element in a bundle\textquotesingle{}s \char`\"{}extensions.\+xml\char`\"{} configuration file. Must be overridden by subclasses.

The \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPoint}{Extension\+Point}} facility entirely leaves it to the \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPoint}{Extension\+Point}} subclass to define the content (child elements and attributes) of the extension element. The only required attribute on the \char`\"{}extension\char`\"{} element is the \char`\"{}point\char`\"{} attribute identifying the extension point (and thus the \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPoint}{Extension\+Point}} object handling the \char`\"{}extension\char`\"{} element.
