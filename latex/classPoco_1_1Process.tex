Creates a new process for the given command and returns a \mbox{\hyperlink{classPoco_1_1ProcessHandle}{Process\+Handle}} of the new process. The given arguments are passed to the command on the command line.

If in\+Pipe, out\+Pipe or err\+Pipe is non-\/null, the corresponding standard input, standard output or standard error stream of the launched process is redirected to the \mbox{\hyperlink{classPoco_1_1Pipe}{Pipe}}. \mbox{\hyperlink{classPoco_1_1PipeInputStream}{Pipe\+Input\+Stream}} or \mbox{\hyperlink{classPoco_1_1PipeOutputStream}{Pipe\+Output\+Stream}} can be used to send receive data from, or send data to the process.

Note\+: the same \mbox{\hyperlink{classPoco_1_1Pipe}{Pipe}} can be used for both out\+Pipe and err\+Pipe.

After a \mbox{\hyperlink{classPoco_1_1Pipe}{Pipe}} has been passed as in\+Pipe, only write operations are valid. After a \mbox{\hyperlink{classPoco_1_1Pipe}{Pipe}} has been passed as out\+Pipe or err\+Pipe, only read operations are valid.

It is forbidden to pass the same pipe as in\+Pipe and out\+Pipe or err\+Pipe.

Creates a new process for the given command and returns a \mbox{\hyperlink{classPoco_1_1ProcessHandle}{Process\+Handle}} of the new process. The given arguments are passed to the command on the command line. The process starts executing in the specified initial directory.

If in\+Pipe, out\+Pipe or err\+Pipe is non-\/null, the corresponding standard input, standard output or standard error stream of the launched process is redirected to the \mbox{\hyperlink{classPoco_1_1Pipe}{Pipe}}. \mbox{\hyperlink{classPoco_1_1PipeInputStream}{Pipe\+Input\+Stream}} or \mbox{\hyperlink{classPoco_1_1PipeOutputStream}{Pipe\+Output\+Stream}} can be used to send receive data from, or send data to the process.

Note\+: the same \mbox{\hyperlink{classPoco_1_1Pipe}{Pipe}} can be used for both out\+Pipe and err\+Pipe.

After a \mbox{\hyperlink{classPoco_1_1Pipe}{Pipe}} has been passed as in\+Pipe, only write operations are valid. After a \mbox{\hyperlink{classPoco_1_1Pipe}{Pipe}} has been passed as out\+Pipe or err\+Pipe, only read operations are valid.

It is forbidden to pass the same pipe as in\+Pipe and out\+Pipe or err\+Pipe.
