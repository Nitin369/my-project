\doxysubsection{Detailed Description}
An \mbox{\hyperlink{classPoco_1_1Util_1_1OptionProcessor}{Option\+Processor}} is used to process the command line arguments of an application.

The \mbox{\hyperlink{classPoco_1_1Util_1_1OptionProcessor_a1ac0c584d04f59d67ba8c68ce4adf54a}{process()}} method takes an argument from the command line. If that argument starts with an option prefix, the argument is further processed. Otherwise, the argument is ignored and false is returned. The argument must match one of the options given in the \mbox{\hyperlink{classPoco_1_1Util_1_1OptionSet}{Option\+Set}} that is passed to the \mbox{\hyperlink{classPoco_1_1Util_1_1OptionProcessor}{Option\+Processor}} with the constructor. If an option is part of a group, at most one option of the group can be passed to the \mbox{\hyperlink{classPoco_1_1Util_1_1OptionProcessor}{Option\+Processor}}. Otherwise an Incompatible\+Options\+Exception is thrown. If the same option is given multiple times, but the option is not repeatable, a Duplicate\+Option\+Exception is thrown. If the option is not recognized, a Unexpected\+Argument\+Exception is thrown. If the option requires an argument, but none is given, an Missing\+Argument\+Exception is thrown. If no argument is expected, but one is present, a Unexpected\+Argument\+Exception is thrown. If a partial option name is ambiguous, an Ambiguous\+Option\+Exception is thrown.

The \mbox{\hyperlink{classPoco_1_1Util_1_1OptionProcessor}{Option\+Processor}} supports two modes\+: Unix mode and default mode. In Unix mode, the option prefix is a dash \textquotesingle{}-\/\textquotesingle{}. A dash must be followed by a short option name, or another dash, followed by a (partial) long option name. In default mode, the option prefix is a slash \textquotesingle{}/\textquotesingle{}, followed by a (partial) long option name. If the special option \textquotesingle{}--\textquotesingle{} is encountered in Unix mode, all following options are ignored.

\mbox{\hyperlink{classPoco_1_1Util_1_1Option}{Option}} arguments can be specified in three ways. If a Unix short option (\char`\"{}-\/o\char`\"{}) is given, the argument directly follows the option name, without any delimiting character or space (\char`\"{}-\/ovalue\char`\"{}). In default option mode, or if a Unix long option (\char`\"{}-\/-\/option\char`\"{}) is given, the option argument is delimited from the option name with either an equal sign (\textquotesingle{}=\textquotesingle{}) or a colon (\textquotesingle{}\+:\textquotesingle{}), as in \char`\"{}-\/-\/option=value\char`\"{} or \char`\"{}/option\+:value\char`\"{}. Finally, a required option argument can be specified on the command line after the option, delimited with a space, as in \char`\"{}-\/-\/option value\char`\"{} or \char`\"{}-\/o value\char`\"{}. The latter only works for required option arguments, not optional ones.

Enables (flag == true) or disables (flag == false) Unix-\/style option processing.

If Unix-\/style processing is enabled, options are expected to begin with a single or a double dash (\textquotesingle{}-\/\textquotesingle{} or \textquotesingle{}--\textquotesingle{}, respectively). A single dash must be followed by a short option name. A double dash must be followed by a (partial) full option name.

If Unix-\/style processing is disabled, options are expected to begin with a slash (\textquotesingle{}/\textquotesingle{}), followed by a (partial) full option name.
