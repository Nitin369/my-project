\doxysubsection{Detailed Description}
This configuration class extracts configuration properties from a \mbox{\hyperlink{namespacePoco_1_1JSON}{J\+S\+ON}} object. An X\+Path-\/like syntax for property names is supported to allow full access to the \mbox{\hyperlink{namespacePoco_1_1JSON}{J\+S\+ON}} object.

Given the following \mbox{\hyperlink{namespacePoco_1_1JSON}{J\+S\+ON}} object as an example\+: \{ \char`\"{}config\char`\"{} \+: \{ \char`\"{}prop1\char`\"{} \+: \char`\"{}value1\char`\"{}, \char`\"{}prop2\char`\"{} \+: 10, \char`\"{}prop3\char`\"{} \+: \mbox{[} \char`\"{}element1\char`\"{}, \char`\"{}element2\char`\"{} \mbox{]}, \char`\"{}prop4\char`\"{} \+: \{ \char`\"{}prop5\char`\"{} \+: false, \char`\"{}prop6\char`\"{} \+: null \} \} \} The following property names would be valid and would yield the shown values\+:

config.\+prop1 -\/-\/$>$ \char`\"{}value1\char`\"{} config.\+prop3\mbox{[}1\mbox{]} -\/-\/$>$ \char`\"{}element2\char`\"{} config.\+prop4.\+prop5 -\/-\/$>$ false

\begin{DoxyItemize}
\item numerical values\+: non zero becomes true, zero becomes false
\item strings\+: true, yes, on become true, false, no, off become false Case does not matter. If the value contains references to other properties (\$\{$<$property$>$\}), these are expanded.
