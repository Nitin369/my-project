\doxysubsection{Detailed Description}
\subsubsection*{template$<$class Service\+Handler$>$\newline
class Poco\+::\+Net\+::\+Socket\+Connector$<$ Service\+Handler $>$}

This class implements the Connector part of the Acceptor-\/\+Connector design pattern.

The Acceptor-\/\+Connector pattern has been described in the book \char`\"{}\+Pattern Languages of Program Design 3\char`\"{}, edited by Robert Martin, Frank Buschmann and Dirk Riehle (Addison Wesley, 1997).

The Acceptor-\/\+Connector design pattern decouples connection establishment and service initialization in a distributed system from the processing performed once a service is initialized. This decoupling is achieved with three components\+: Acceptors, Connectors and Service Handlers. The Connector actively establishes a connection with a remote server socket (usually managed by an Acceptor) and initializes a Service Handler to manage the connection.

The \mbox{\hyperlink{classPoco_1_1Net_1_1SocketConnector}{Socket\+Connector}} sets up a \mbox{\hyperlink{classPoco_1_1Net_1_1StreamSocket}{Stream\+Socket}}, initiates a non-\/blocking connect operation and registers itself for \mbox{\hyperlink{classPoco_1_1Net_1_1ReadableNotification}{Readable\+Notification}}, \mbox{\hyperlink{classPoco_1_1Net_1_1WritableNotification}{Writable\+Notification}} and \mbox{\hyperlink{classPoco_1_1Net_1_1ErrorNotification}{Error\+Notification}}. \mbox{\hyperlink{classPoco_1_1Net_1_1ReadableNotification}{Readable\+Notification}} or \mbox{\hyperlink{classPoco_1_1Net_1_1WritableNotification}{Writable\+Notification}} denote the successful establishment of the connection.

When the \mbox{\hyperlink{classPoco_1_1Net_1_1StreamSocket}{Stream\+Socket}} becomes readable or writeable, the \mbox{\hyperlink{classPoco_1_1Net_1_1SocketConnector}{Socket\+Connector}} creates a Service\+Handler to service the connection and unregisters itself.

In case of an error (\mbox{\hyperlink{classPoco_1_1Net_1_1ErrorNotification}{Error\+Notification}}), the \mbox{\hyperlink{classPoco_1_1Net_1_1SocketConnector}{Socket\+Connector}} unregisters itself and calls the \mbox{\hyperlink{classPoco_1_1Net_1_1SocketConnector_a47a147e5e311816f8d387cf3111a05a2}{on\+Error()}} method, which can be overridden by subclasses to perform custom error handling.

The Service\+Handler class must provide a constructor that takes a \mbox{\hyperlink{classPoco_1_1Net_1_1StreamSocket}{Stream\+Socket}} and a \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor}{Socket\+Reactor}} as arguments, e.\+g.\+: My\+Service\+Handler(const Stream\+Socket\& socket, Service\+Reactor\& reactor)

When the Service\+Handler is done, it must destroy itself.

Subclasses can override the \mbox{\hyperlink{classPoco_1_1Net_1_1SocketConnector_a5d73c2ca2b15c754a4672d2b5c107bf5}{create\+Service\+Handler()}} factory method if special steps are necessary to create a Service\+Handler object.
