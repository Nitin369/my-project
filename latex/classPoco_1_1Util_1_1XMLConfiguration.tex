\doxysubsection{Detailed Description}
This configuration class extracts configuration properties from an \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} document. An X\+Path-\/like syntax for property names is supported to allow full access to the \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} document. \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} namespaces are not supported. The name of the root element of the \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} document is not significant and ignored. Periods in tag names are not supported.

Given the following \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} document as an example\+: \begin{DoxyVerb}<config>
    <prop1>value1</prop1>
    <prop2>value2</prop2>
    <prop3>
       <prop4 attr="value3"/>
       <prop4 attr="value4"/>
    </prop3>
    <prop5 id="first">value5</prop5>
    <prop5 id="second">value6</prop5>
</config>
\end{DoxyVerb}


The following property names would be valid and would yield the shown values\+: \begin{DoxyVerb}prop1                 -> value1
prop2                 -> value2
prop3.prop4           -> (empty string)
prop3.prop4[@attr]    -> value3
prop3.prop4[1][@attr] -> value4
prop5[0]              -> value5
prop5[1]              -> value6
prop5[@id=first]      -> value5
prop5[@id='second']   -> value6
\end{DoxyVerb}


Enumerating attributes is not supported. Calling keys(\char`\"{}prop3.\+prop4\char`\"{}) will return an empty range.

As a special feature, the delimiter character used to delimit property names can be changed to something other than period (\textquotesingle{}.\textquotesingle{}) by passing the desired character to the constructor. This allows working with \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} documents having element names with periods in them.
