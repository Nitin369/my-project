\doxysubsection{Detailed Description}
A \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1TCP_1_1Timer}{Timer}} allows to schedule tasks (\mbox{\hyperlink{classPoco_1_1RemotingNG_1_1TCP_1_1TimerTask}{Timer\+Task}} objects) for future execution in a background thread. Tasks may be scheduled for one-\/time execution, or for repeated execution at regular intervals.

The \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1TCP_1_1Timer}{Timer}} object creates a thread that executes all scheduled tasks sequentially. Therefore, tasks should complete their work as quickly as possible, otherwise subsequent tasks may be delayed.

\mbox{\hyperlink{classPoco_1_1RemotingNG_1_1TCP_1_1Timer}{Timer}} is save for multithreaded use -\/ multiple threads can schedule new tasks simultaneously.

This is a simplified version of the \mbox{\hyperlink{classPoco_1_1Util_1_1Timer}{Poco\+::\+Util\+::\+Timer}} class.

Schedules a task for periodic execution at a fixed rate.

The task is first executed after the given delay. Subsequently, the task is executed periodically every number of milliseconds specified by interval.

If task execution takes longer than the given interval, further executions are delayed.

If a task is currently running, it is allowed to finish.

\mbox{\hyperlink{classPoco_1_1Task}{Task}} cancellation is done asynchronously. If wait is false, \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1TCP_1_1Timer_a83d103286fa65c6238741e181637ff9a}{cancel()}} returns immediately and the task queue will be purged as soon as the currently running task finishes. If wait is true, waits until the queue has been purged.
