\doxysubsection{Detailed Description}
This class implements the client-\/side of a H\+T\+T\+PS session.

To send a H\+T\+T\+PS request to a H\+T\+T\+PS server, first instantiate a \mbox{\hyperlink{classPoco_1_1Net_1_1HTTPSClientSession}{H\+T\+T\+P\+S\+Client\+Session}} object and specify the server\textquotesingle{}s host name and port number.

Then create a \mbox{\hyperlink{classPoco_1_1Net_1_1HTTPRequest}{H\+T\+T\+P\+Request}} object, fill it accordingly, and pass it as argument to the \mbox{\hyperlink{classPoco_1_1Net_1_1HTTPClientSession_a1739c6ac53078ebf6aaf4bf6cfab5c5a}{send\+Request()}} method.

\mbox{\hyperlink{classPoco_1_1Net_1_1HTTPClientSession_a1739c6ac53078ebf6aaf4bf6cfab5c5a}{send\+Request()}} will return an output stream that can be used to send the request body, if there is any.

After you are done sending the request body, create a \mbox{\hyperlink{classPoco_1_1Net_1_1HTTPResponse}{H\+T\+T\+P\+Response}} object and pass it to \mbox{\hyperlink{classPoco_1_1Net_1_1HTTPClientSession_a43b4f42c34a2d9b3ed3263745645f99a}{receive\+Response()}}.

This will return an input stream that can be used to read the response body.

See R\+FC 2616 \href{http://www.faqs.org/rfcs/rfc2616.html}{\texttt{ http\+://www.\+faqs.\+org/rfcs/rfc2616.\+html}} for more information about the H\+T\+TP protocol.

Note that sending requests that neither contain a content length field in the header nor are using chunked transfer encoding will result in a S\+SL protocol violation, as the framework shuts down the socket after sending the message body. No orderly S\+SL shutdown will be performed in this case.

If session caching has been enabled for the \mbox{\hyperlink{classPoco_1_1Net_1_1Context}{Context}} object passed to the \mbox{\hyperlink{classPoco_1_1Net_1_1HTTPSClientSession}{H\+T\+T\+P\+S\+Client\+Session}}, the \mbox{\hyperlink{classPoco_1_1Net_1_1HTTPSClientSession}{H\+T\+T\+P\+S\+Client\+Session}} class will attempt to reuse a previously obtained \mbox{\hyperlink{classPoco_1_1Net_1_1Session}{Session}} object in case of a reconnect.
