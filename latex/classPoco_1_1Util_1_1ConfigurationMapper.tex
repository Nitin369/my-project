\doxysubsection{Detailed Description}
This configuration maps a property hierarchy into another hierarchy.

For example, given a configuration with the following properties\+: config.\+value1 config.\+value2 config.\+sub.\+value1 config.\+sub.\+value2 and a \mbox{\hyperlink{classPoco_1_1Util_1_1ConfigurationView}{Configuration\+View}} with from\+Prefix == \char`\"{}config\char`\"{} and to\+Prefix == \char`\"{}root.\+conf\char`\"{}, then the above properties will be available via the mapper as root.\+conf.\+value1 root.\+conf.\+value2 root.\+conf.\+sub.\+value1 root.\+conf.\+sub.\+value2

From\+Prefix can be empty, in which case, and given to\+Prefix == \char`\"{}root\char`\"{}, the properties will be available as root.\+config.\+value1 root.\+config.\+value2 root.\+config.\+sub.\+value1 root.\+config.\+sub.\+value2

This is equivalent to the functionality of the \mbox{\hyperlink{classPoco_1_1Util_1_1ConfigurationView}{Configuration\+View}} class.

Similarly, to\+Prefix can also be empty. Given from\+Prefix == \char`\"{}config\char`\"{} and to\+Prefix == \char`\"{}\char`\"{}, the properties will be available as value1 value2 sub.\+value1 sub.\+value2

If both from\+Prefix and to\+Prefix are empty, no mapping is performed.

A \mbox{\hyperlink{classPoco_1_1Util_1_1ConfigurationMapper}{Configuration\+Mapper}} is most useful in combination with a \mbox{\hyperlink{classPoco_1_1Util_1_1LayeredConfiguration}{Layered\+Configuration}}.
