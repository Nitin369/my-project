\doxysubsection{Detailed Description}
A \mbox{\hyperlink{classPoco_1_1OSP_1_1Web_1_1WebSession}{Web\+Session}} is used for tracking users between different H\+T\+T\+P(\+S) requests. \mbox{\hyperlink{classPoco_1_1OSP_1_1Web_1_1WebSession}{Web\+Session}} objects are managed by a session manager (usually, an implementation of the \mbox{\hyperlink{classPoco_1_1OSP_1_1Web_1_1WebSessionService}{Web\+Session\+Service}}).

The \mbox{\hyperlink{classPoco_1_1OSP_1_1Web_1_1WebSession}{Web\+Session}} can also be used to store arbitrary data (attributes) in the form of key-\/value pairs between different requests. \mbox{\hyperlink{classPoco_1_1Any}{Poco\+::\+Any}} is used for storing values, so practically any object can be attached to a session.

A \mbox{\hyperlink{classPoco_1_1OSP_1_1Web_1_1WebSession}{Web\+Session}} has a time out. If a \mbox{\hyperlink{classPoco_1_1OSP_1_1Web_1_1WebSession}{Web\+Session}} instance is not accessed for a given time, it will be destroyed by the session manager.

Note that the iterator-\/based access methods (\mbox{\hyperlink{classPoco_1_1OSP_1_1Web_1_1WebSession_a7c7f23cd2b4e815223769d1f0cb3ce74}{find()}}, \mbox{\hyperlink{classPoco_1_1OSP_1_1Web_1_1WebSession_aab64741851e47c424dac5028c1b08ed9}{begin()}}, \mbox{\hyperlink{classPoco_1_1OSP_1_1Web_1_1WebSession_a1491d721fedd4d9c9da1b459bcafab8c}{end()}}) are not thread-\/safe.

When a new session is created, the session manager will generate a C\+S\+RF synchronizer token, which can be used in forms to prevent Cross-\/\+Site Request Forgery (C\+S\+RF) attacks. The name of the property is \char`\"{}\#csrf\+Token\char`\"{}. It can also be obtained by calling the \mbox{\hyperlink{classPoco_1_1OSP_1_1Web_1_1WebSession_a59ed22b85f830031ce4b6e0059e7e19f}{csrf\+Token()}} member function.
