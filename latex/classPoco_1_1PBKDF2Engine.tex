\doxysubsection{Detailed Description}
\subsubsection*{template$<$class P\+RF$>$\newline
class Poco\+::\+P\+B\+K\+D\+F2\+Engine$<$ P\+R\+F $>$}

This class implementes the Password-\/\+Based Key Derivation Function 2, as specified in R\+FC 2898. The underlying \mbox{\hyperlink{classPoco_1_1DigestEngine}{Digest\+Engine}} (\mbox{\hyperlink{classPoco_1_1HMACEngine}{H\+M\+A\+C\+Engine}}, etc.), which must accept the passphrase as constructor argument (std\+::string), must be given as template argument.

P\+B\+K\+D\+F2 (Password-\/\+Based Key Derivation Function 2) is a key derivation function that is part of R\+SA Laboratories\textquotesingle{} Public-\/\+Key Cryptography Standards (P\+K\+CS) series, specifically P\+K\+CS \#5 v2.\+0, also published as Internet Engineering \mbox{\hyperlink{classPoco_1_1Task}{Task}} Force\textquotesingle{}s R\+FC 2898. It replaces an earlier standard, P\+B\+K\+D\+F1, which could only produce derived keys up to 160 bits long.

P\+B\+K\+D\+F2 applies a pseudorandom function, such as a cryptographic hash, cipher, or H\+M\+AC to the input password or passphrase along with a salt value and repeats the process many times to produce a derived key, which can then be used as a cryptographic key in subsequent operations. The added computational work makes password cracking much more difficult, and is known as key stretching. When the standard was written in 2000, the recommended minimum number of iterations was 1000, but the parameter is intended to be increased over time as C\+PU speeds increase. Having a salt added to the password reduces the ability to use precomputed hashes (rainbow tables) for attacks, and means that multiple passwords have to be tested individually, not all at once. The standard recommends a salt length of at least 64 bits. \mbox{[}Wikipedia\mbox{]}

The P\+B\+K\+D\+F2 algorithm is implemented as a \mbox{\hyperlink{classPoco_1_1DigestEngine}{Digest\+Engine}}. The passphrase is specified by calling \mbox{\hyperlink{classPoco_1_1DigestEngine_ac3de0beba3214e7c41b97ceab478f4d1}{update()}}.

Example (W\+P\+A2)\+: \mbox{\hyperlink{classPoco_1_1PBKDF2Engine}{P\+B\+K\+D\+F2\+Engine}}$<$H\+M\+A\+C\+Engine$<$\+S\+H\+A1\+Engine$>$ $>$ pbkdf2(ssid, 4096, 256); pbkdf2.\+update(passphrase); \mbox{\hyperlink{classPoco_1_1DigestEngine_a8e6e74183768a08e8f00112e5c6c055e}{Digest\+Engine\+::\+Digest}} d = pbkdf2.\+digest();
