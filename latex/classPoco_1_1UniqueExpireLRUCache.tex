\subsubsection*{template$<$class T\+Key, class T\+Value, class T\+Mutex = Fast\+Mutex, class T\+Event\+Mutex = Fast\+Mutex$>$\newline
class Poco\+::\+Unique\+Expire\+L\+R\+U\+Cache$<$ T\+Key, T\+Value, T\+Mutex, T\+Event\+Mutex $>$}

A \mbox{\hyperlink{classPoco_1_1UniqueExpireLRUCache}{Unique\+Expire\+L\+R\+U\+Cache}} combines L\+RU caching and time based per entry expire caching. One can define for each cache entry a seperate timepoint but also limit the size of the cache (per default\+: 1024). Each T\+Value object must thus offer the following method\+:

const \mbox{\hyperlink{classPoco_1_1Timestamp}{Poco\+::\+Timestamp}}\& get\+Expiration() const;

which returns the absolute timepoint when the entry will be invalidated. Accessing an object will N\+OT update this absolute expire timepoint. You can use the \mbox{\hyperlink{classPoco_1_1ExpirationDecorator}{Poco\+::\+Expiration\+Decorator}} to add the get\+Expiration method to values that do not have a get\+Expiration function.
