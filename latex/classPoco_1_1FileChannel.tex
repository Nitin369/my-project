\doxysubsection{Detailed Description}
A \mbox{\hyperlink{classPoco_1_1Channel}{Channel}} that writes to a file. This class supports flexible log file rotation and archiving, as well as automatic purging of archived log files.

Only the message\textquotesingle{}s text is written, followed by a newline.

Chain this channel to a \mbox{\hyperlink{classPoco_1_1FormattingChannel}{Formatting\+Channel}} with an appropriate \mbox{\hyperlink{classPoco_1_1Formatter}{Formatter}} to control what is in the text.

The \mbox{\hyperlink{classPoco_1_1FileChannel}{File\+Channel}} support log file rotation based on log file size or time intervals. Archived log files can be compressed in gzip format. Older archived files can be automatically deleted (purged).

The rotation strategy can be specified with the \char`\"{}rotation\char`\"{} property, which can take one of the follwing values\+:


\begin{DoxyItemize}
\item never\+: no log rotation
\item \mbox{[}day,\mbox{]}\mbox{[}hh\mbox{]}\+:mm\+: the file is rotated on specified day/time day -\/ day is specified as long or short day name (Monday$\vert$\+Mon, Tuesday$\vert$\+Tue, ... ); day can be omitted, in which case log is rotated every day hh -\/ valid hour range is 00-\/23; hour can be omitted, in which case log is rotated every hour mm -\/ valid minute range is 00-\/59; minute must be specified
\item daily\+: the file is rotated daily
\item weekly\+: the file is rotated every seven days
\item monthly\+: the file is rotated every 30 days
\item $<$n$>$ minutes\+: the file is rotated every $<$n$>$ minutes, where $<$n$>$ is an integer greater than zero.
\item $<$n$>$ hours\+: the file is rotated every $<$n$>$ hours, where $<$n$>$ is an integer greater than zero.
\item $<$n$>$ days\+: the file is rotated every $<$n$>$ days, where $<$n$>$ is an integer greater than zero.
\item $<$n$>$ weeks\+: the file is rotated every $<$n$>$ weeks, where $<$n$>$ is an integer greater than zero.
\item $<$n$>$ months\+: the file is rotated every $<$n$>$ months, where $<$n$>$ is an integer greater than zero and a month has 30 days.
\item $<$n$>$\+: the file is rotated when its size exceeds $<$n$>$ bytes.
\item $<$n$>$ K\+: the file is rotated when its size exceeds $<$n$>$ Kilobytes.
\item $<$n$>$ M\+: the file is rotated when its size exceeds $<$n$>$ Megabytes.
\end{DoxyItemize}

N\+O\+TE\+: For periodic log file rotation (daily, weekly, monthly, etc.), the date and time of log file creation or last rotation is written into the first line of the log file. This is because there is no reliable way to find out the real creation date of a file on many platforms (e.\+g., most Unix platforms do not provide the creation date, and Windows has its own issues with its \char`\"{}\+File System Tunneling Capabilities\char`\"{}).

Using the \char`\"{}archive\char`\"{} property it is possible to specify how archived log files are named. The following values for the \char`\"{}archive\char`\"{} property are supported\+:


\begin{DoxyItemize}
\item number\+: A number, starting with 0, is appended to the name of archived log files. The newest archived log file always has the number 0. For example, if the log file is named \char`\"{}access.\+log\char`\"{}, and it fulfils the criteria for rotation, the file is renamed to \char`\"{}access.\+log.\+0\char`\"{}. If a file named \char`\"{}access.\+log.\+0\char`\"{} already exists, it is renamed to \char`\"{}access.\+log.\+1\char`\"{}, and so on.
\item timestamp\+: A timestamp is appended to the log file name. For example, if the log file is named \char`\"{}access.\+log\char`\"{}, and it fulfils the criteria for rotation, the file is renamed to \char`\"{}access.\+log.\+20050802110300\char`\"{}.
\end{DoxyItemize}

Using the \char`\"{}times\char`\"{} property it is possible to specify time mode for the day/time based rotation. The following values for the \char`\"{}times\char`\"{} property are supported\+:


\begin{DoxyItemize}
\item utc\+: Rotation strategy is based on U\+TC time (default).
\item local\+: Rotation strategy is based on local time.
\end{DoxyItemize}

Archived log files can be compressed using the gzip compression method. Compressing can be controlled with the \char`\"{}compress\char`\"{} property. The following values for the \char`\"{}compress\char`\"{} property are supported\+:


\begin{DoxyItemize}
\item true\+: Compress archived log files.
\item false\+: Do not compress archived log files.
\end{DoxyItemize}

Archived log files can be automatically purged, either if they reach a certain age, or if the number of archived log files reaches a given maximum number. This is controlled by the purge\+Age and purge\+Count properties.

The purge\+Age property can have the following values\+:


\begin{DoxyItemize}
\item $<$n$>$ \mbox{[}seconds\mbox{]}\+: the maximum age is $<$n$>$ seconds.
\item $<$n$>$ minutes\+: the maximum age is $<$n$>$ minutes.
\item $<$n$>$ hours\+: the maximum age is $<$n$>$ hours.
\item $<$n$>$ days\+: the maximum age is $<$n$>$ days.
\item $<$n$>$ weeks\+: the maximum age is $<$n$>$ weeks.
\item $<$n$>$ months\+: the maximum age is $<$n$>$ months, where a month has 30 days.
\end{DoxyItemize}

The purge\+Count property has an integer value that specifies the maximum number of archived log files. If the number is exceeded, archived log files are deleted, starting with the oldest. When \char`\"{}none\char`\"{} or empty string are supplied, they reset purge\+Count to none (no purging).

The flush property specifies whether each log message is flushed immediately to the log file (which may hurt application performance, but ensures that everything is in the log in case of a system crash), Valid values are\+: \begin{DoxyVerb}  * true:  Every essages is immediately flushed to the log file (default).
  * false: Messages are not immediately flushed to the log file.

The rotateOnOpen property specifies whether an existing log file should be 
rotated (and archived) when the channel is opened. Valid values are:

  * true:  The log file is rotated (and archived) when the channel is opened.
  * false: Log messages will be appended to an existing log file,
           if it exists (unless other conditions for a rotation are met). 
           This is the default.

For a more lightweight file channel class, see SimpleFileChannel. 
\end{DoxyVerb}
