\doxysubsection{Detailed Description}
\subsubsection*{template$<$class Base$>$\newline
class Poco\+::\+Class\+Loader$<$ Base $>$}

The \mbox{\hyperlink{classPoco_1_1ClassLoader}{Class\+Loader}} loads C++ classes from shared libraries at runtime. It must be instantiated with a root class of the loadable classes. For a class to be loadable from a library, the library must provide a \mbox{\hyperlink{classPoco_1_1Manifest}{Manifest}} of all the classes it contains. The \mbox{\hyperlink{classPoco_1_1Manifest}{Manifest}} for a shared library can be easily built with the help of the macros in the header file \char`\"{}\+Foundation/\+Class\+Library.\+h\char`\"{}.

Starting with P\+O\+CO release 1.\+3, a class library can export multiple manifests. In addition to the default (unnamed) manifest, multiple named manifests can be exported, each having a different base class.

There is one important restriction\+: one instance of \mbox{\hyperlink{classPoco_1_1ClassLoader}{Class\+Loader}} can only load one manifest from a class library.
