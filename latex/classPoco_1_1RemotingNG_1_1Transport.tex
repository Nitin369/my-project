\doxysubsection{Detailed Description}
\mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport}{Transport}} objects are responsible for setting up and maintaining a network (or other kind of) connection between \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Proxy}{Proxy}} objects on the client side and \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Listener}{Listener}} objects on the server side, and for exchanging messages over that connection.

A \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport}{Transport}} object also maintains \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Serializer}{Serializer}} and \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Deserializer}{Deserializer}} objects.

\mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport}{Transport}} objects must support two message exchange patterns (M\+EP)\+:
\begin{DoxyItemize}
\item One-\/\+Way\+: a message is sent from the client to the server and the client does not expect to receive a reply from the server. With this pattern, the server has no way to provide status information back to the client, and the client has no way to know whether the call was successful.
\item Request-\/\+Reply\+: a message is sent from the client to the server and the client expects to receive a reply from the server. The reply may be a regular reply containing a return value and/or output parameters, or it may be a fault message, reporting an exception to the client.
\end{DoxyItemize}

Provided a \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport}{Transport}} object is already connected, sending a one-\/way message is a three-\/step process\+:
\begin{DoxyEnumerate}
\item A new one-\/way message is started with a call to \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_a15503615c6875f4a62257792e6e9288e}{begin\+Message()}}. This will prepare the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport}{Transport}} to deliver a one-\/way message to the server, as well as setting up a \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Serializer}{Serializer}} for creating the one-\/way message.
\item The \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Serializer}{Serializer}} returned by \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_a15503615c6875f4a62257792e6e9288e}{begin\+Message()}} is used to compose the actual message.
\item The one-\/way message is delivered to the server with a call to \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_aa6d52470d41cd5745fd6718ab3b410ad}{send\+Message()}}. This completes the message exchange.
\end{DoxyEnumerate}

Doing a request-\/reply message exchange is a five-\/step process.
\begin{DoxyEnumerate}
\item A new request message is started with a call to \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_ab3d447faa49ee0fd0368a0410be164e7}{begin\+Request()}}. This will prepare the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport}{Transport}} to deliver a request message, as well as setting up a \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Serializer}{Serializer}} for creating the request message.
\item The \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Serializer}{Serializer}} returned by \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_ab3d447faa49ee0fd0368a0410be164e7}{begin\+Request()}} is used to compose the actual message.
\item The request is delivered to the server with a call to \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_acfb2df40ad48ef04c43997a193e5208f}{send\+Request()}}. After delivering the request, \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_acfb2df40ad48ef04c43997a193e5208f}{send\+Request()}} will wait for a reply from the server, and set up a \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Deserializer}{Deserializer}} for reading the reply.
\item The reply message is read using the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Deserializer}{Deserializer}} returned by \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_acfb2df40ad48ef04c43997a193e5208f}{send\+Request()}}.
\item A call to \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_a2c14a664920b61048982d1c779bcb76a}{end\+Request()}} ends the message exchange.
\end{DoxyEnumerate}

\mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport}{Transport}} objects must be able to deal with incomplete message exchange sequences. For example, if an exception occurs between a call to \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_ab3d447faa49ee0fd0368a0410be164e7}{begin\+Request()}} and the corresponding call to \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_acfb2df40ad48ef04c43997a193e5208f}{send\+Request()}}, \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_acfb2df40ad48ef04c43997a193e5208f}{send\+Request()}} and \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_a2c14a664920b61048982d1c779bcb76a}{end\+Request()}} will not be called for this exchange. When \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_ab3d447faa49ee0fd0368a0410be164e7}{begin\+Request()}} or \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport_a15503615c6875f4a62257792e6e9288e}{begin\+Message()}} is called the next time, it must do the necessary cleanup to reset the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport}{Transport}} object to a well-\/defined state.
