\doxysubsection{Detailed Description}
The \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPointService}{Extension\+Point\+Service}} allows a bundle to define extension points (or \char`\"{}hooks\char`\"{}) where other bundles can \char`\"{}hook in\char`\"{} functionality, thus extending the bundle.

The service listens for E\+V\+\_\+\+B\+U\+N\+D\+L\+E\+\_\+\+S\+T\+A\+R\+T\+ED events. For every started bundle, the service tries to retrieve the bundle\textquotesingle{}s \char`\"{}extensions.\+xml\char`\"{} file. If the file exists, the file is parsed using a D\+OM parser.

The \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPointService}{Extension\+Point\+Service}} then iterates over all \char`\"{}extension\char`\"{} elements in that document. Every \char`\"{}extension\char`\"{} element must specify an attribute named \char`\"{}point\char`\"{} that specifies the \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPoint}{Extension\+Point}} object to invoke for this element. The service then invokes the \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPoint_a667347045bd8acccd6fac30092c33e1a}{Extension\+Point\+::handle\+Extension()}} member function with the bundle and the \char`\"{}extension\char`\"{} element as arguments.

The \char`\"{}extensions.\+xml\char`\"{} file basically looks as follows\+: $<$extensions$>$ $<$extension point=\char`\"{}$<$id$>$\char`\"{}$>$

$<$/extension$>$

$<$/extensions$>$

The service name of the \mbox{\hyperlink{classPoco_1_1OSP_1_1ExtensionPointService}{Extension\+Point\+Service}} is \char`\"{}osp.\+core.\+xp\char`\"{}.
