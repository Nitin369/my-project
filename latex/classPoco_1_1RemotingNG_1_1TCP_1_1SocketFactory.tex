\doxysubsection{Detailed Description}
\mbox{\hyperlink{classPoco_1_1RemotingNG_1_1TCP_1_1SocketFactory}{Socket\+Factory}} is used by the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1TCP_1_1ConnectionManager}{Connection\+Manager}} to create a socket for the connection to the listener on the server.

The default implementation simply creates a \mbox{\hyperlink{classPoco_1_1Net_1_1StreamSocket}{Poco\+::\+Net\+::\+Stream\+Socket}}. If a secure connection should be created, \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1TCP_1_1SocketFactory_acfa9ba2ed82ec6cf6b5b9bc6436251e6}{create\+Socket()}} should be overridden in a subclass, creating a \mbox{\hyperlink{classPoco_1_1Net_1_1SecureStreamSocket}{Poco\+::\+Net\+::\+Secure\+Stream\+Socket}} instead.

The Remoting NG \mbox{\hyperlink{namespacePoco_1_1RemotingNG_1_1TCP}{T\+CP}} transport uses the \char`\"{}remoting.\+tcp\char`\"{} \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}} scheme for unencrypted socket connections and the \char`\"{}remoting.\+tcps\char`\"{} \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}} scheme for S\+S\+L/\+T\+LS connections. Subclasses supporting S\+S\+L/\+T\+LS should check the \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}} scheme and create an appropriate socket.
