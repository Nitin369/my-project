\doxysubsection{Detailed Description}
The \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1EventDispatcher}{Event\+Dispatcher}} is responsible for delivering events fired by service objects to remote subscribers.

\mbox{\hyperlink{classPoco_1_1Event}{Event}} dispatching and delivery in \mbox{\hyperlink{namespacePoco_1_1RemotingNG}{Remoting\+NG}} works as follows\+: If a service object has events, the corresponding \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1RemoteObject}{Remote\+Object}} class will have a delegate member function for every service object event, as well as providing all the events itself (inherited from the interface class). During construction of the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1RemoteObject}{Remote\+Object}}, its delegates will be bound to the service objects\textquotesingle{} events. The delegates will then simply fire the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1RemoteObject}{Remote\+Object}}\textquotesingle{}s event, when invoked.

The \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1EventDispatcher}{Event\+Dispatcher}} subclass for a specific service will have delegate functions for all service object events, which will be bound to the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1RemoteObject}{Remote\+Object}}\textquotesingle{}s events. The \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1EventDispatcher}{Event\+Dispatcher}} class maintains a list of event subscribers. When an event is fired, the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1EventDispatcher}{Event\+Dispatcher}}\textquotesingle{}s delegate will dispatch the event to all registered event subscribers, using a \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Transport}{Transport}}. This works in a very similar way to a \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Proxy}{Proxy}}.

Registers a remote \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1EventSubscriber}{Event\+Subscriber}} identified by the given subscriber\+U\+RI. Events will be sent to the specified endpoint, which must be a proper \mbox{\hyperlink{namespacePoco_1_1RemotingNG}{Remoting\+NG}} \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}}. Depending on the actual transport-\/specific events implementation, subscriber\+U\+RI and endpoint\+U\+RI may be the same or different.

If a non-\/null expire\+Time is given, the subscription will automatically expire at the time given in expire\+Time.

If a subscription with the given subscriber\+U\+RI already exists, the expire\+Time of the existing subscription will be updated.
