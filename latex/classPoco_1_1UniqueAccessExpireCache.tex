\subsubsection*{template$<$class T\+Key, class T\+Value, class T\+Mutex = Fast\+Mutex, class T\+Event\+Mutex = Fast\+Mutex$>$\newline
class Poco\+::\+Unique\+Access\+Expire\+Cache$<$ T\+Key, T\+Value, T\+Mutex, T\+Event\+Mutex $>$}

An \mbox{\hyperlink{classPoco_1_1UniqueAccessExpireCache}{Unique\+Access\+Expire\+Cache}} caches entries for a given time span. In contrast to \mbox{\hyperlink{classPoco_1_1ExpireCache}{Expire\+Cache}} which only allows to set a per cache expiration value, it allows to define expiration per Cache\+Entry. Each T\+Value object must thus offer the following method\+:

const \mbox{\hyperlink{classPoco_1_1Timespan}{Poco\+::\+Timespan}}\& get\+Timeout() const;

which returns the relative timespan for how long the entry should be valid without being accessed! The absolute expire timepoint is calculated as \mbox{\hyperlink{namespacePoco_1_1Data_1_1Keywords_a839e06ffbdee1328f646e4c83e0ae1fd}{now()}} + get\+Timeout(). Accessing an object will update this absolute expire timepoint. You can use the \mbox{\hyperlink{classPoco_1_1AccessExpirationDecorator}{Poco\+::\+Access\+Expiration\+Decorator}} to add the get\+Expiration method to values that do not have a get\+Expiration function.

Be careful when using an \mbox{\hyperlink{classPoco_1_1UniqueAccessExpireCache}{Unique\+Access\+Expire\+Cache}}. A cache is often used like cache.\+has(x) followed by cache.\+get x). Note that it could happen that the \char`\"{}has\char`\"{} call works, then the current execution thread gets descheduled, time passes, the entry gets invalid, thus leading to an empty \mbox{\hyperlink{classPoco_1_1SharedPtr}{Shared\+Ptr}} being returned when \char`\"{}get\char`\"{} is invoked.
