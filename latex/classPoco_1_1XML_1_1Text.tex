\doxysubsection{Detailed Description}
The \mbox{\hyperlink{classPoco_1_1XML_1_1Text}{Text}} interface inherits from \mbox{\hyperlink{classPoco_1_1XML_1_1CharacterData}{Character\+Data}} and represents the textual content (termed character data in \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}}) of an \mbox{\hyperlink{classPoco_1_1XML_1_1Element}{Element}} or \mbox{\hyperlink{classPoco_1_1XML_1_1Attr}{Attr}}. If there is no markup inside an element\textquotesingle{}s content, the text is contained in a single object implementing the \mbox{\hyperlink{classPoco_1_1XML_1_1Text}{Text}} interface that is the only child of the element. If there is markup, it is parsed into the information items (elements, comments, etc.) and \mbox{\hyperlink{classPoco_1_1XML_1_1Text}{Text}} nodes that form the list of children of the element.

When a document is first made available via the D\+OM, there is only one \mbox{\hyperlink{classPoco_1_1XML_1_1Text}{Text}} node for each block of text. Users may create adjacent \mbox{\hyperlink{classPoco_1_1XML_1_1Text}{Text}} nodes that represent the contents of a given element without any intervening markup, but should be aware that there is no way to represent the separations between these nodes in \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} or H\+T\+ML, so they will not (in general) persist between D\+OM editing sessions. The \mbox{\hyperlink{classPoco_1_1XML_1_1AbstractNode_a0fbcc44acbff9b146285d325af4f6d18}{normalize()}} method on \mbox{\hyperlink{classPoco_1_1XML_1_1Element}{Element}} merges any such adjacent \mbox{\hyperlink{classPoco_1_1XML_1_1Text}{Text}} objects into a single node for each block of text.

A new node of the same type, which is inserted as the next sibling of this node, contains all the content at and after the offset point. When the offset is equal to the length of this node, the new node has no data.
