\doxysubsection{Detailed Description}
This class, which is part of the Reactor pattern, implements the \char`\"{}\+Initiation Dispatcher\char`\"{}.

The Reactor pattern has been described in the book \char`\"{}\+Pattern Languages of Program Design\char`\"{} by Jim Coplien and Douglas C. Schmidt (Addison Wesley, 1995).

The Reactor design pattern handles service requests that are delivered concurrently to an application by one or more clients. Each service in an application may consist of several methods and is represented by a separate event handler. The event handler is responsible for servicing service-\/specific requests. The \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor}{Socket\+Reactor}} dispatches the event handlers.

\mbox{\hyperlink{classPoco_1_1Event}{Event}} handlers (any class can be an event handler -\/ there is no base class for event handlers) can be registered with the \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor_af6d366d0de6f903709f61a07e21ab508}{add\+Event\+Handler()}} method and deregistered with the \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor_a2c239f3cedcfb75af342c231bf7e8f9d}{remove\+Event\+Handler()}} method.

An event handler is always registered for a certain socket, which is given in the call to \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor_af6d366d0de6f903709f61a07e21ab508}{add\+Event\+Handler()}}. \mbox{\hyperlink{classPoco_1_1Any}{Any}} method of the event handler class can be registered to handle the event -\/ the only requirement is that the method takes a pointer to an instance of \mbox{\hyperlink{classPoco_1_1Net_1_1SocketNotification}{Socket\+Notification}} (or a subclass of it) as argument.

Once started, the \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor}{Socket\+Reactor}} waits for events on the registered sockets, using \mbox{\hyperlink{classPoco_1_1Net_1_1Socket_a4b64fe1fc3848e6dec346b49f0708248}{Socket\+::select()}}. If an event is detected, the corresponding event handler is invoked. There are five event types (and corresponding notification classes) defined\+: \mbox{\hyperlink{classPoco_1_1Net_1_1ReadableNotification}{Readable\+Notification}}, \mbox{\hyperlink{classPoco_1_1Net_1_1WritableNotification}{Writable\+Notification}}, \mbox{\hyperlink{classPoco_1_1Net_1_1ErrorNotification}{Error\+Notification}}, \mbox{\hyperlink{classPoco_1_1Net_1_1TimeoutNotification}{Timeout\+Notification}}, \mbox{\hyperlink{classPoco_1_1Net_1_1IdleNotification}{Idle\+Notification}} and \mbox{\hyperlink{classPoco_1_1Net_1_1ShutdownNotification}{Shutdown\+Notification}}.

The \mbox{\hyperlink{classPoco_1_1Net_1_1ReadableNotification}{Readable\+Notification}} will be dispatched if a socket becomes readable. The \mbox{\hyperlink{classPoco_1_1Net_1_1WritableNotification}{Writable\+Notification}} will be dispatched if a socket becomes writable. The \mbox{\hyperlink{classPoco_1_1Net_1_1ErrorNotification}{Error\+Notification}} will be dispatched if there is an error condition on a socket.

If the timeout expires and no event has occurred, a \mbox{\hyperlink{classPoco_1_1Net_1_1TimeoutNotification}{Timeout\+Notification}} will be dispatched to all event handlers registered for it. This is done in the \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor_af776c3f9c6205e278c4c522c98b7da27}{on\+Timeout()}} method which can be overridden by subclasses to perform custom timeout processing.

If there are no sockets for the \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor}{Socket\+Reactor}} to pass to \mbox{\hyperlink{classPoco_1_1Net_1_1Socket_a4b64fe1fc3848e6dec346b49f0708248}{Socket\+::select()}}, an \mbox{\hyperlink{classPoco_1_1Net_1_1IdleNotification}{Idle\+Notification}} will be dispatched to all event handlers registered for it. This is done in the \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor_a12ff2423749461fce52913012383f923}{on\+Idle()}} method which can be overridden by subclasses to perform custom idle processing. Since \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor_a12ff2423749461fce52913012383f923}{on\+Idle()}} will be called repeatedly in a loop, it is recommended to do a short sleep or yield in the event handler.

Finally, when the \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor}{Socket\+Reactor}} is about to shut down (as a result of \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor_aa63b954aded0ade13d59e60aa8241492}{stop()}} being called), it dispatches a \mbox{\hyperlink{classPoco_1_1Net_1_1ShutdownNotification}{Shutdown\+Notification}} to all event handlers. This is done in the \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor_a1270a2f51648df02d8f698c311421213}{on\+Shutdown()}} method which can be overridded by subclasses to perform custom shutdown processing.

The \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor}{Socket\+Reactor}} is implemented so that it can run in its own thread. It is also possible to run multiple Socket\+Reactors in parallel, as long as they work on different sockets.

It is safe to call \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor_af6d366d0de6f903709f61a07e21ab508}{add\+Event\+Handler()}} and \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor_a2c239f3cedcfb75af342c231bf7e8f9d}{remove\+Event\+Handler()}} from another thread while the \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor}{Socket\+Reactor}} is running. Also, it is safe to call \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor_af6d366d0de6f903709f61a07e21ab508}{add\+Event\+Handler()}} and \mbox{\hyperlink{classPoco_1_1Net_1_1SocketReactor_a2c239f3cedcfb75af342c231bf7e8f9d}{remove\+Event\+Handler()}} from event handlers.
