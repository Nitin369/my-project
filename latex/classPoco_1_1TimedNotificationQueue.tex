\doxysubsection{Detailed Description}
A \mbox{\hyperlink{classPoco_1_1TimedNotificationQueue}{Timed\+Notification\+Queue}} object provides a way to implement timed, asynchronous notifications. This is especially useful for sending notifications from one thread to another, for example from a background thread to the main (user interface) thread.

The \mbox{\hyperlink{classPoco_1_1TimedNotificationQueue}{Timed\+Notification\+Queue}} is quite similar to the \mbox{\hyperlink{classPoco_1_1NotificationQueue}{Notification\+Queue}} class. The only difference to \mbox{\hyperlink{classPoco_1_1NotificationQueue}{Notification\+Queue}} is that each \mbox{\hyperlink{classPoco_1_1Notification}{Notification}} is tagged with a \mbox{\hyperlink{classPoco_1_1Timestamp}{Timestamp}}. When inserting a \mbox{\hyperlink{classPoco_1_1Notification}{Notification}} into the queue, the \mbox{\hyperlink{classPoco_1_1Notification}{Notification}} is inserted according to the given \mbox{\hyperlink{classPoco_1_1Timestamp}{Timestamp}}, with lower \mbox{\hyperlink{classPoco_1_1Timestamp}{Timestamp}} values being inserted before higher ones.

Notifications are dequeued in order of their timestamps.

\mbox{\hyperlink{classPoco_1_1TimedNotificationQueue}{Timed\+Notification\+Queue}} has some restrictions regarding multithreaded use. While multiple threads may enqueue notifications, only one thread at a time may dequeue notifications from the queue.

If two threads try to dequeue a notification simultaneously, the results are undefined.
