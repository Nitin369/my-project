\doxysubsection{Detailed Description}
This interface represents an entity, either parsed or unparsed, in an \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} document. Note that this models the entity itself not the entity declaration. \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}} declaration modeling has been left for a later Level of the D\+OM specification.

The node\+Name attribute that is inherited from \mbox{\hyperlink{classPoco_1_1XML_1_1Node}{Node}} contains the name of the entity.

An \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} processor may choose to completely expand entities before the structure model is passed to the D\+OM; in this case there will be no \mbox{\hyperlink{classPoco_1_1XML_1_1EntityReference}{Entity\+Reference}} nodes in the document tree.

\mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} does not mandate that a non-\/validating \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} processor read and process entity declarations made in the external subset or declared in external parameter entities. This means that parsed entities declared in the external subset need not be expanded by some classes of applications, and that the replacement value of the entity may not be available. When the replacement value is available, the corresponding \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}} node\textquotesingle{}s child list represents the structure of that replacement text. Otherwise, the child list is empty.

The resolution of the children of the \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}} (the replacement value) may be lazily evaluated; actions by the user (such as calling the child\+Nodes method on the \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}} \mbox{\hyperlink{classPoco_1_1XML_1_1Node}{Node}}) are assumed to trigger the evaluation.

The D\+OM Level 1 does not support editing \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}} nodes; if a user wants to make changes to the contents of an \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}}, every related \mbox{\hyperlink{classPoco_1_1XML_1_1EntityReference}{Entity\+Reference}} node has to be replaced in the structure model by a clone of the \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}}\textquotesingle{}s contents, and then the desired changes must be made to each of those clones instead. \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}} nodes and all their descendants are readonly.

An \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}} node does not have any parent.
