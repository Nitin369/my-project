\doxysubsection{Detailed Description}
The \mbox{\hyperlink{classPoco_1_1Util_1_1Application}{Application}} class implements the main subsystem in a process. The application class is responsible for initializing all its subsystems.

Subclasses can and should override the following virtual methods\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{classPoco_1_1Util_1_1Application_a82f62471a68493049a7d3955f34976ca}{initialize()}} (the one-\/argument, protected variant)
\item \mbox{\hyperlink{classPoco_1_1Util_1_1Application_ae441fc171418cf80cc88922cae09ce8f}{uninitialize()}}
\item \mbox{\hyperlink{classPoco_1_1Util_1_1Application_acf6923ae08f4345e928871b87deed928}{reinitialize()}}
\item \mbox{\hyperlink{classPoco_1_1Util_1_1Application_a387af69805eec896f1938c79167186dd}{define\+Options()}}
\item \mbox{\hyperlink{classPoco_1_1Util_1_1Application_a0c5eb94bff2ba0576f59be0995cbe509}{handle\+Option()}}
\item \mbox{\hyperlink{classPoco_1_1Util_1_1Application_acb65f34a4ebed4fa8b9987904a2555fc}{main()}}
\end{DoxyItemize}

The application\textquotesingle{}s main logic should be implemented in the \mbox{\hyperlink{classPoco_1_1Util_1_1Application_acb65f34a4ebed4fa8b9987904a2555fc}{main()}} method.

There may be at most one instance of the \mbox{\hyperlink{classPoco_1_1Util_1_1Application}{Application}} class in a process.

The \mbox{\hyperlink{classPoco_1_1Util_1_1Application}{Application}} class maintains a \mbox{\hyperlink{classPoco_1_1Util_1_1LayeredConfiguration}{Layered\+Configuration}} (available via the \mbox{\hyperlink{classPoco_1_1Util_1_1Application_a4c619b854bdce613aec5fd6a991fb529}{config()}} member function) consisting of\+:
\begin{DoxyItemize}
\item a \mbox{\hyperlink{classPoco_1_1Util_1_1MapConfiguration}{Map\+Configuration}} (priority -\/100) storing application-\/specific properties, as well as properties from bound command line arguments.
\item a \mbox{\hyperlink{classPoco_1_1Util_1_1SystemConfiguration}{System\+Configuration}} (priority 100)
\item the configurations loaded with \mbox{\hyperlink{classPoco_1_1Util_1_1Application_a11e1d403f05c0f980d24f9555c8777b9}{load\+Configuration()}}.
\end{DoxyItemize}

The \mbox{\hyperlink{classPoco_1_1Util_1_1Application}{Application}} class sets a few default properties in its configuration. These are\+:
\begin{DoxyItemize}
\item application.\+path\+: the absolute path to application executable
\item application.\+name\+: the file name of the application executable
\item application.\+base\+Name\+: the file name (excluding extension) of the application executable
\item application.\+dir\+: the path to the directory where the application executable resides
\item application.\+config\+Dir\+: the path to the directory where the last configuration file loaded with \mbox{\hyperlink{classPoco_1_1Util_1_1Application_a11e1d403f05c0f980d24f9555c8777b9}{load\+Configuration()}} was found.
\end{DoxyItemize}

If \mbox{\hyperlink{classPoco_1_1Util_1_1Application_a11e1d403f05c0f980d24f9555c8777b9}{load\+Configuration()}} has never been called, application.\+config\+Dir will be equal to application.\+dir.

The P\+O\+C\+O\+\_\+\+A\+P\+P\+\_\+\+M\+A\+IN macro can be used to implement main(argc, argv). If P\+O\+CO has been built with P\+O\+C\+O\+\_\+\+W\+I\+N32\+\_\+\+U\+T\+F8, P\+O\+C\+O\+\_\+\+A\+P\+P\+\_\+\+M\+A\+IN supports \mbox{\hyperlink{classPoco_1_1Unicode}{Unicode}} command line arguments.

Loads configuration information from a default location.

The configuration(s) will be added to the application\textquotesingle{}s \mbox{\hyperlink{classPoco_1_1Util_1_1LayeredConfiguration}{Layered\+Configuration}} with the given priority.

The configuration file(s) must be located in the same directory as the executable or a parent directory of it, and must have the same base name as the executable, with one of the following extensions\+: .properties, .ini or .xml.

The .properties file, if it exists, is loaded first, followed by the .ini file and the .xml file.

If the application is built in debug mode (the \+\_\+\+D\+E\+B\+UG preprocessor macro is defined) and the base name of the appication executable ends with a \textquotesingle{}d\textquotesingle{}, a config file without the \textquotesingle{}d\textquotesingle{} ending its base name is also found.

Example\+: Given the application \char`\"{}\+Sample\+Appd.\+exe\char`\"{}, built in debug mode. Then \mbox{\hyperlink{classPoco_1_1Util_1_1Application_a11e1d403f05c0f980d24f9555c8777b9}{load\+Configuration()}} will automatically find a configuration file named \char`\"{}\+Sample\+App.\+properties\char`\"{} if it exists and if \char`\"{}\+Sample\+Appd.\+properties\char`\"{} cannot be found.

Returns the number of configuration files loaded, which may be zero.
