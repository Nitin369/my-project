\subsubsection*{template$<$class T\+Key, class T\+Value, class T\+Mutex = Fast\+Mutex, class T\+Event\+Mutex = Fast\+Mutex$>$\newline
class Poco\+::\+Expire\+Cache$<$ T\+Key, T\+Value, T\+Mutex, T\+Event\+Mutex $>$}

An \mbox{\hyperlink{classPoco_1_1ExpireCache}{Expire\+Cache}} caches entries for a fixed time period (per default 10 minutes). Entries expire independently of the access pattern, i.\+e. after a constant time. If you require your objects to expire after they were not accessed for a given time period use a \mbox{\hyperlink{classPoco_1_1AccessExpireCache}{Poco\+::\+Access\+Expire\+Cache}}.

Be careful when using an \mbox{\hyperlink{classPoco_1_1ExpireCache}{Expire\+Cache}}. A cache is often used like cache.\+has(x) followed by cache.\+get x). Note that it could happen that the \char`\"{}has\char`\"{} call works, then the current execution thread gets descheduled, time passes, the entry gets invalid, thus leading to an empty \mbox{\hyperlink{classPoco_1_1SharedPtr}{Shared\+Ptr}} being returned when \char`\"{}get\char`\"{} is invoked.
