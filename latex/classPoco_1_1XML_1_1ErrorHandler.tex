\doxysubsection{Detailed Description}
If a S\+AX application needs to implement customized error handling, it must implement this interface and then register an instance with the \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} reader using the set\+Error\+Handler method. The parser will then report all errors and warnings through this interface.

W\+A\+R\+N\+I\+NG\+: If an application does not register an \mbox{\hyperlink{classPoco_1_1XML_1_1ErrorHandler}{Error\+Handler}}, \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} parsing errors will go unreported, except that S\+A\+X\+Parse\+Exceptions will be thrown for fatal errors. In order to detect validity errors, an \mbox{\hyperlink{classPoco_1_1XML_1_1ErrorHandler}{Error\+Handler}} that does something with \mbox{\hyperlink{classPoco_1_1XML_1_1ErrorHandler_a86da5a0c23660a5ce964abc009e69142}{error()}} calls must be registered.

For \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} processing errors, a S\+AX driver must use this interface in preference to throwing an exception\+: it is up to the application to decide whether to throw an exception for different types of errors and warnings. Note, however, that there is no requirement that the parser continue to report additional errors after a call to fatal\+Error. In other words, a S\+AX driver class may throw an exception after reporting any fatal\+Error. Also parsers may throw appropriate exceptions for non-\/\+X\+ML errors. For example, \mbox{\hyperlink{classPoco_1_1XML_1_1XMLReader_aafa45ffeb18d9d9ebe9c7859bb413a37}{X\+M\+L\+Reader\+::parse()}} would throw an I\+O\+Exception for errors accessing entities or the document.

Receive notification of a recoverable error.

This corresponds to the definition of \char`\"{}error\char`\"{} in section 1.\+2 of the W3C \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} 1.\+0 Recommendation. For example, a validating parser would use this callback to report the violation of a validity constraint. The default behaviour is to take no action.

The S\+AX parser must continue to provide normal parsing events after invoking this method\+: it should still be possible for the application to process the document through to the end. If the application cannot do so, then the parser should report a fatal error even if the \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} recommendation does not require it to do so.

Filters may use this method to report other, non-\/\+X\+ML errors as well.
