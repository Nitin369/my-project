\doxysubsection{Detailed Description}
This is an optional extension handler for S\+A\+X2 to provide lexical information about an \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} document, such as comments and C\+D\+A\+TA section boundaries. \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} readers are not required to recognize this handler, and it is not part of core-\/only S\+A\+X2 distributions.

The events in the lexical handler apply to the entire document, not just to the document element, and all lexical handler events must appear between the content handler\textquotesingle{}s start\+Document and end\+Document events.

To set the \mbox{\hyperlink{classPoco_1_1XML_1_1LexicalHandler}{Lexical\+Handler}} for an \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} reader, use the set\+Property method with the property name \href{http://xml.org/sax/properties/lexical-handler}{\texttt{ http\+://xml.\+org/sax/properties/lexical-\/handler}} and an object implementing this interface (or null) as the value. If the reader does not report lexical events, it will throw a S\+A\+X\+Not\+Recognized\+Exception when you attempt to register the handler.

Report the start of D\+TD declarations, if any.

This method is intended to report the beginning of the D\+O\+C\+T\+Y\+PE declaration; if the document has no D\+O\+C\+T\+Y\+PE declaration, this method will not be invoked.

All declarations reported through \mbox{\hyperlink{classPoco_1_1XML_1_1DTDHandler}{D\+T\+D\+Handler}} or \mbox{\hyperlink{classPoco_1_1XML_1_1DeclHandler}{Decl\+Handler}} events must appear between the start\+D\+TD and end\+D\+TD events. Declarations are assumed to belong to the internal D\+TD subset unless they appear between start\+Entity and end\+Entity events. Comments and processing instructions from the D\+TD should also be reported between the start\+D\+TD and end\+D\+TD events, in their original order of (logical) occurrence; they are not required to appear in their correct locations relative to \mbox{\hyperlink{classPoco_1_1XML_1_1DTDHandler}{D\+T\+D\+Handler}} or \mbox{\hyperlink{classPoco_1_1XML_1_1DeclHandler}{Decl\+Handler}} events, however.

Note that the start/end\+D\+TD events will appear within the start/end\+Document events from \mbox{\hyperlink{classPoco_1_1XML_1_1ContentHandler}{Content\+Handler}} and before the first start\+Element event.

Report the beginning of some internal and external \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} entities.

The reporting of parameter entities (including the external D\+TD subset) is optional, and S\+A\+X2 drivers that report \mbox{\hyperlink{classPoco_1_1XML_1_1LexicalHandler}{Lexical\+Handler}} events may not implement it; you can use the \href{http://xml.org/sax/features/lexical-handler/parameter-entities}{\texttt{ http\+://xml.\+org/sax/features/lexical-\/handler/parameter-\/entities}} feature to query or control the reporting of parameter entities.

General entities are reported with their regular names, parameter entities have \textquotesingle{}\textquotesingle{} prepended to their names, and the external D\+TD subset has the pseudo-\/entity name \char`\"{}\mbox{[}dtd\mbox{]}\char`\"{}.

When a S\+A\+X2 driver is providing these events, all other events must be properly nested within start/end entity events. There is no additional requirement that events from \mbox{\hyperlink{classPoco_1_1XML_1_1DeclHandler}{Decl\+Handler}} or \mbox{\hyperlink{classPoco_1_1XML_1_1DTDHandler}{D\+T\+D\+Handler}} be properly ordered.

Note that skipped entities will be reported through the skipped\+Entity event, which is part of the \mbox{\hyperlink{classPoco_1_1XML_1_1ContentHandler}{Content\+Handler}} interface.

Because of the streaming event model that S\+AX uses, some entity boundaries cannot be reported under any circumstances\+: \begin{DoxyVerb}* general entities within attribute values
* parameter entities within declarations
\end{DoxyVerb}


These will be silently expanded, with no indication of where the original entity boundaries were.

Note also that the boundaries of character references (which are not really entities anyway) are not reported.

All start/end\+Entity events must be properly nested. ~\newline
