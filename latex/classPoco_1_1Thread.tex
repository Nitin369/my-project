\doxysubsection{Detailed Description}
This class implements a platform-\/independent wrapper to an operating system thread.

Every \mbox{\hyperlink{classPoco_1_1Thread}{Thread}} object gets a unique (within its process) numeric thread ID. Furthermore, a thread can be assigned a name. The name of a thread can be changed at any time.

Function returns true if sleep attempt was completed, false if sleep was interrupted by a \mbox{\hyperlink{classPoco_1_1Thread_aadac087896a05ef0887131364a68a0f9}{wake\+Up()}} call. A frequent scenario where \mbox{\hyperlink{classPoco_1_1Thread_a42034bed42eb25ceb60ee39da88216b4}{try\+Sleep()}}/wake\+Up() pair of functions is useful is with threads spending most of the time idle, with periodic activity between the idle times; trying to sleep (as opposed to sleeping) allows immediate ending of idle thread from the outside.

The \mbox{\hyperlink{classPoco_1_1Thread_a42034bed42eb25ceb60ee39da88216b4}{try\+Sleep()}} and \mbox{\hyperlink{classPoco_1_1Thread_aadac087896a05ef0887131364a68a0f9}{wake\+Up()}} calls should be used with understanding that the suspended state is not a true sleep, but rather a state of waiting for an event, with timeout expiration. This makes order of calls significant; calling \mbox{\hyperlink{classPoco_1_1Thread_aadac087896a05ef0887131364a68a0f9}{wake\+Up()}} before calling \mbox{\hyperlink{classPoco_1_1Thread_a42034bed42eb25ceb60ee39da88216b4}{try\+Sleep()}} will prevent the next ~\newline
