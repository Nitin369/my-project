\doxysubsection{Detailed Description}
A \mbox{\hyperlink{classPoco_1_1Net_1_1InvalidCertificateHandler}{Invalid\+Certificate\+Handler}} is invoked whenever an error occurs verifying the certificate. It allows the user to inspect and accept/reject the certificate. One can install one\textquotesingle{}s own \mbox{\hyperlink{classPoco_1_1Net_1_1InvalidCertificateHandler}{Invalid\+Certificate\+Handler}} by implementing this interface. Note that in the implementation file of the subclass the following code must be present (assuming you use the namespace My\+\_\+\+A\+PI and the name of your handler class is My\+Gui\+Handler)\+:

\#include \char`\"{}\+Poco/\+Net/\+Certificate\+Handler\+Factory.\+h\char`\"{} ... \mbox{\hyperlink{CertificateHandlerFactory_8h_a10e019d3d96d67083465daeef5a54d29}{P\+O\+C\+O\+\_\+\+R\+E\+G\+I\+S\+T\+E\+R\+\_\+\+C\+H\+F\+A\+C\+T\+O\+R\+Y(\+My\+\_\+\+A\+P\+I, My\+Gui\+Handler)}}

One can either set the handler directly in the startup code of the main method of ones application by calling

\mbox{\hyperlink{classPoco_1_1Net_1_1SSLManager_aaee0b196d700365f00e49645c0de7db1}{S\+S\+L\+Manager\+::instance()}}.initialize(mypassphrase\+Handler, mygui\+Handler, my\+S\+S\+L\+Context)

or in case one uses \mbox{\hyperlink{classPoco_1_1Util_1_1Application}{Poco\+::\+Util\+::\+Application}} one can rely on an \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} configuration and put the following entry under the path open\+S\+S\+L.\+invalid\+Certificate\+Handler\+:

$<$invalid\+Certificate\+Handler$>$ $<$name$>$My\+Gui\+Handler$<$name$>$ $<$options$>$ \mbox{[}...\mbox{]} // Put optional config params for the handler here $<$/options$>$ $<$/invalid\+Certificate\+Handler$>$

Note that the name of the \mbox{\hyperlink{classPoco_1_1Net_1_1InvalidCertificateHandler}{Invalid\+Certificate\+Handler}} must be same as the one provided to the P\+O\+C\+O\+\_\+\+R\+E\+G\+I\+S\+T\+E\+R\+\_\+\+C\+H\+F\+A\+C\+T\+O\+RY macro.
