\doxysubsection{Detailed Description}
This class serializes S\+A\+X2 \mbox{\hyperlink{classPoco_1_1XML_1_1ContentHandler}{Content\+Handler}}, \mbox{\hyperlink{classPoco_1_1XML_1_1LexicalHandler}{Lexical\+Handler}} and \mbox{\hyperlink{classPoco_1_1XML_1_1DTDHandler}{D\+T\+D\+Handler}} events back into a stream.

Various consistency checks are performed on the written data (i.\+e. there must be exactly one root element and every \mbox{\hyperlink{classPoco_1_1XML_1_1XMLWriter_a255af0fba72b490c0c40cde5baed4e62}{start\+Element()}} must have a matching \mbox{\hyperlink{classPoco_1_1XML_1_1XMLWriter_a1f02caaa8d7d9274b0d53108af3763c6}{end\+Element()}}).

The \mbox{\hyperlink{classPoco_1_1XML_1_1XMLWriter}{X\+M\+L\+Writer}} supports optional pretty-\/printing of the serialized \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}}. Note, however, that pretty-\/printing \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} data alters the information set of the document being written, since in \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} all whitespace is potentially relevant to an application.

The writer contains extensive support for \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} Namespaces, so that a client ~\newline
 application does not have to keep track of prefixes and supply xmlns attributes.

If the client does not provide namespace prefixes (either by specifying them as part of the qualified name given to \mbox{\hyperlink{classPoco_1_1XML_1_1XMLWriter_a255af0fba72b490c0c40cde5baed4e62}{start\+Element()}}, or by calling \mbox{\hyperlink{classPoco_1_1XML_1_1XMLWriter_a4785d4b9081cb898b06770a76a38e55a}{start\+Prefix\+Mapping()}}), the \mbox{\hyperlink{classPoco_1_1XML_1_1XMLWriter}{X\+M\+L\+Writer}} automatically generates namespace prefixes in the form ns1, ns2, etc.
