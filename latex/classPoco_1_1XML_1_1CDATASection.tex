\doxysubsection{Detailed Description}
C\+D\+A\+TA sections are used to escape blocks of text containing characters that would otherwise be regarded as markup. The only delimiter that is recognized in a C\+D\+A\+TA section is the \char`\"{}\mbox{]}\mbox{]}$>$\char`\"{} string that ends the C\+D\+A\+TA section. C\+D\+A\+TA sections cannot be nested. Their primary purpose is for including material such as \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} fragments, without needing to escape all the delimiters.

The D\+O\+M\+String attribute of the \mbox{\hyperlink{classPoco_1_1XML_1_1Text}{Text}} node holds the text that is contained by the C\+D\+A\+TA section. Note that this may contain characters that need to be escaped outside of C\+D\+A\+TA sections and that, depending on the character encoding (\char`\"{}charset\char`\"{}) chosen for serialization, it may be impossible to write out some characters as part of a C\+D\+A\+TA section.

The \mbox{\hyperlink{classPoco_1_1XML_1_1CDATASection}{C\+D\+A\+T\+A\+Section}} interface inherits from the \mbox{\hyperlink{classPoco_1_1XML_1_1CharacterData}{Character\+Data}} interface through the \mbox{\hyperlink{classPoco_1_1XML_1_1Text}{Text}} interface. Adjacent \mbox{\hyperlink{classPoco_1_1XML_1_1CDATASection}{C\+D\+A\+T\+A\+Section}} nodes are not merged by use of the normalize method on the \mbox{\hyperlink{classPoco_1_1XML_1_1Element}{Element}} interface.

Note\+: Because no markup is recognized within a \mbox{\hyperlink{classPoco_1_1XML_1_1CDATASection}{C\+D\+A\+T\+A\+Section}}, character numeric references cannot be used as an escape mechanism when serializing. Therefore, action needs to be taken when serializing a \mbox{\hyperlink{classPoco_1_1XML_1_1CDATASection}{C\+D\+A\+T\+A\+Section}} with a character encoding where some of the contained characters cannot be represented. Failure to do so would not produce well-\/formed \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}}. One potential solution in the serialization process is to end the C\+D\+A\+TA section before the character, output the character using a character reference or entity reference, and open a new C\+D\+A\+TA section for any further characters in the text node. Note, however, that some code conversion libraries at the time of writing do not return an error or exception when a character is missing from the encoding, making the task of ensuring that data is not corrupted on serialization more difficult.
