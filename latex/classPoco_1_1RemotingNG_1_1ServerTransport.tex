\doxysubsection{Detailed Description}
\mbox{\hyperlink{classPoco_1_1RemotingNG_1_1ServerTransport}{Server\+Transport}} objects are used as mediators between a \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Listener}{Listener}}, receiving a request, and the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1ORB}{O\+RB}}, \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Skeleton}{Skeleton}} and Method\+Handlers, processing the request.

The \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1ServerTransport}{Server\+Transport}} allows the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Skeleton}{Skeleton}} to deserialize the request message in order to determine the method name, select an appropriate \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1MethodHandler}{Method\+Handler}}, based on the method name and pass control to the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1MethodHandler}{Method\+Handler}}. The \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1MethodHandler}{Method\+Handler}} then uses the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1ServerTransport}{Server\+Transport}} to send back a response message.

Verify that the request is properly authenticated.

This method is called if the method or its class has the @authenticated attribute set to true, or if it has a @permission attribute set to a non-\/empty permission.

An implementation should check whether the current request contains valid authentication information. Typically, an implementation will obtain the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Credentials}{Credentials}} object for the request, obtain the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Authenticator}{Authenticator}} from the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Listener}{Listener}}, and call its \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1ServerTransport_aef991a3ff983af8cc3ecf85746732c38}{authenticate()}} method to check the credentials.

Returns true if the request is properly authenticated, otherwise false.

The default implementation simply returns true.

Verify that an authenticated user has the given permission, which is required to invoke the given method.

The given permission will be the the one specified with the @permission attribute for the respective remote method. If no @permission attribute has been specified for a remote method, or the permission is an empty string, this method will not be called.

An implementation should obtain the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Authorizer}{Authorizer}} object from the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Listener}{Listener}}, and call the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Authorizer}{Authorizer}}\textquotesingle{}s \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1ServerTransport_ad113a897a8c83d224850730e080e52b1}{authorize()}} method to authorize the method call.

Returns true if authorization is successful, otherwise false.

The default implementation simply returns true.
