\doxysubsection{Detailed Description}
This configuration implements a \char`\"{}view\char`\"{} into a sub-\/hierarchy of another configuration.

For example, given a configuration with the following properties\+: config.\+value1 config.\+value2 config.\+sub.\+value1 config.\+sub.\+value2 and a \mbox{\hyperlink{classPoco_1_1Util_1_1ConfigurationView}{Configuration\+View}} with the prefix \char`\"{}config\char`\"{}, then the above properties will be available via the view as value1 value2 sub.\+value1 sub.\+value2

A \mbox{\hyperlink{classPoco_1_1Util_1_1ConfigurationView}{Configuration\+View}} is most useful in combination with a \mbox{\hyperlink{classPoco_1_1Util_1_1LayeredConfiguration}{Layered\+Configuration}}.

If a property is not found in the view, it is searched in the original configuration. Given the above example configuration, the property named \char`\"{}config.\+value1\char`\"{} will still be found in the view.

The main reason for this is that placeholder expansion (\$\{property\}) still works as expected given a \mbox{\hyperlink{classPoco_1_1Util_1_1ConfigurationView}{Configuration\+View}}.
