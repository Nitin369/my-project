\doxysubsection{Detailed Description}
This subsystem implementation sets up the \mbox{\hyperlink{namespacePoco_1_1OSP}{O\+SP}} runtime environment, including \mbox{\hyperlink{classPoco_1_1OSP_1_1BundleLoader}{Bundle\+Loader}}, \mbox{\hyperlink{classPoco_1_1OSP_1_1CodeCache}{Code\+Cache}} and \mbox{\hyperlink{classPoco_1_1OSP_1_1ServiceRegistry}{Service\+Registry}}.

It also defines the \char`\"{}clean\char`\"{} command line option, which forces the code cache to be cleared, and the \char`\"{}bundles\char`\"{} command line option which allows for specification of additional bundle repositories.

The following configuration properties are used\+:
\begin{DoxyItemize}
\item osp.\+bundle\+Repository\+: list of directories containing bundles (defaults to \$\{application.\+dir\}bundles)
\item osp.\+code\+Cache\+: location of code cache directory (defaults to \$\{application.\+dir\}code\+Cache)
\item osp.\+auto\+Update\+Code\+Cache\+: automatically update shared libraries in code\+Cache with newer versions found in bundles
\item osp.\+shared\+Code\+Cache\+: allow using the same code cache directory for multiple processes, causing certain operations to be guarded by a global lock
\item osp.\+language\+: language used for localization (overrides the system default)
\item osp.\+data the directory where temporary and persistent data for bundles is stored (defaults to \$\{application.\+dir\}data)
\end{DoxyItemize}

The following configuration properties are set\+:
\begin{DoxyItemize}
\item osp.\+version\+: \mbox{\hyperlink{namespacePoco_1_1OSP}{O\+SP}} \mbox{\hyperlink{classPoco_1_1OSP_1_1Version}{Version}} from osp.\+core bundle (only if osp.\+core bundle is present)
\item osp.\+osname\+: name of operating system
\item osp.\+osarch\+: hardware architecture identifier 
\end{DoxyItemize}
