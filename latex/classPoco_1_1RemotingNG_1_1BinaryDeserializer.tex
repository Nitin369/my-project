\doxysubsection{Detailed Description}
A generic \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Deserializer}{Deserializer}} using a proprietary binary serialization format based on the \mbox{\hyperlink{classPoco_1_1BinaryReader}{Poco\+::\+Binary\+Reader}} class. The \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1BinaryDeserializer}{Binary\+Deserializer}} is used in conjunction with the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1BinarySerializer}{Binary\+Serializer}} class.

Please see the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1BinarySerializer}{Binary\+Serializer}} class for more information.

For X\+M\+L-\/based transports, this method allows for deserialization of data either as element content (default) or attribute value.

The rule that deserializers follow is to first call \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Deserializer_a2807dcb2238dc1f27e63164a3ac99ffd}{push\+Attribute()}} for every variable to be deserialized as attribute, then call \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1BinaryDeserializer_a9093f8ec658ca3616bbf29c682405af5}{deserialize\+Message\+Begin()}} or deserialize\+Struct\+Start() for the element containing the attributes.

Example\+: The following code\+: \begin{DoxyVerb}ser.pushAttribute("", "attr1", true);
ser.pushAttribute("", "attr2", true);
ser.deserializeStructBegin("complexType", true);
ser.deserialize("attr1", attr1, true); 
ser.deserialize("attr2", attr2, true); 
ser.deserialize("variable", variable, true); 
ser.deserializeStructEnd("complexType"); 
\end{DoxyVerb}


will read this \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} fragment\+: $<$complex\+Type attr1=\char`\"{}foo\char`\"{} attr2=\char`\"{}bar\char`\"{}$>$$<$variable$>$42$<$/variable$>$...$<$/complex\+Type$>$

Should be overridden by subclasses supporting attributes. The default implementation does nothing.
