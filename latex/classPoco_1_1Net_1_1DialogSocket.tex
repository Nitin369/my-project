\doxysubsection{Detailed Description}
\mbox{\hyperlink{classPoco_1_1Net_1_1DialogSocket}{Dialog\+Socket}} is a subclass of \mbox{\hyperlink{classPoco_1_1Net_1_1StreamSocket}{Stream\+Socket}} that can be used for implementing request-\/response based client server connections.

A request is always a single-\/line command terminated by C\+R-\/\+LF.

A response can either be a single line of text terminated by C\+R-\/\+LF, or multiple lines of text in the format used by the F\+TP and S\+M\+TP protocols.

Limited support for the T\+E\+L\+N\+ET protocol (R\+FC 854) is available.

Warning\+: Do not call \mbox{\hyperlink{classPoco_1_1Net_1_1StreamSocket_a33cee8a58a5b1a47570617d3ea59e5ba}{receive\+Bytes()}} on a \mbox{\hyperlink{classPoco_1_1Net_1_1DialogSocket}{Dialog\+Socket}}. Due to internal buffering in \mbox{\hyperlink{classPoco_1_1Net_1_1DialogSocket}{Dialog\+Socket}}, \mbox{\hyperlink{classPoco_1_1Net_1_1StreamSocket_a33cee8a58a5b1a47570617d3ea59e5ba}{receive\+Bytes()}} may return an unexpected result and interfere with \mbox{\hyperlink{classPoco_1_1Net_1_1DialogSocket}{Dialog\+Socket}}\textquotesingle{}s buffering. Use \mbox{\hyperlink{classPoco_1_1Net_1_1DialogSocket_a9f9789032d7abaf37f88c66c9cadaf5f}{receive\+Raw\+Bytes()}} instead.

Receives a single-\/line or multi-\/line response from the socket connection. The format must be according to one of the response formats specified in the F\+TP (R\+FC 959) or S\+M\+TP (R\+FC 2821) specifications.

The first line starts with a 3-\/digit status code. Following the status code is either a space character (\textquotesingle{} \textquotesingle{} ) (in case of a single-\/line response) or a minus character (\textquotesingle{}-\/\textquotesingle{}) in case of a multi-\/line response. The following lines can have a three-\/digit status code followed by a minus-\/sign and some text, or some arbitrary text only. The last line again begins with a three-\/digit status code (which must be the same as the one in the first line), followed by a space and some arbitrary text. All lines must be terminated by a C\+R-\/\+LF sequence.

The response contains all response lines, separated by a newline character, including the status code. The status code is returned.
