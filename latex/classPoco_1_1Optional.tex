\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename C$>$\newline
class Poco\+::\+Optional$<$ C $>$}

\mbox{\hyperlink{classPoco_1_1Optional}{Optional}} is a simple wrapper class for value types that allows to introduce a specified/unspecified state to value objects.

An \mbox{\hyperlink{classPoco_1_1Optional}{Optional}} can be default constructed. In this case, the \mbox{\hyperlink{classPoco_1_1Optional}{Optional}} will have a Null value and \mbox{\hyperlink{classPoco_1_1Optional_aa976774a4bcc0efe46d7abb7e970b40a}{is\+Specified()}} will return false. Calling \mbox{\hyperlink{classPoco_1_1Optional_a69bcb485f299bfb15877c04fc28c5ce1}{value()}}(without default value) on a Null object will throw a Null\+Value\+Exception.

An \mbox{\hyperlink{classPoco_1_1Optional}{Optional}} can also be constructed from a value. It is possible to assign a value to an \mbox{\hyperlink{classPoco_1_1Optional}{Optional}}, and to reset an \mbox{\hyperlink{classPoco_1_1Optional}{Optional}} to contain a Null value by calling \mbox{\hyperlink{classPoco_1_1Optional_a66683a23049ac6758ba3a4803c07f32a}{clear()}}.

For use with \mbox{\hyperlink{classPoco_1_1Optional}{Optional}}, the value type should support default construction.

Note that the \mbox{\hyperlink{classPoco_1_1Optional}{Optional}} class is basically the same as \mbox{\hyperlink{classPoco_1_1Nullable}{Nullable}}. However, serializers may treat \mbox{\hyperlink{classPoco_1_1Nullable}{Nullable}} and \mbox{\hyperlink{classPoco_1_1Optional}{Optional}} differently. An example is \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} serialization based on \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} Schema, where \mbox{\hyperlink{classPoco_1_1Optional}{Optional}} would be used for an element with min\+Occurs == 0, whereas \mbox{\hyperlink{classPoco_1_1Nullable}{Nullable}} would be used on an element with nillable == true.
