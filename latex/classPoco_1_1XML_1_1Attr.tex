\doxysubsection{Detailed Description}
The \mbox{\hyperlink{classPoco_1_1XML_1_1Attr}{Attr}} interface represents an attribute in an \mbox{\hyperlink{classPoco_1_1XML_1_1Element}{Element}} object. Typically the allowable values for the attribute are defined in a document type definition.

\mbox{\hyperlink{classPoco_1_1XML_1_1Attr}{Attr}} objects inherit the \mbox{\hyperlink{classPoco_1_1XML_1_1Node}{Node}} interface, but since they are not actually child nodes of the element they describe, the D\+OM does not consider them part of the document tree. Thus, the \mbox{\hyperlink{classPoco_1_1XML_1_1Node}{Node}} attributes parent\+Node, previous\+Sibling, and next\+Sibling have a null value for \mbox{\hyperlink{classPoco_1_1XML_1_1Attr}{Attr}} objects. The D\+OM takes the view that attributes are properties of elements rather than having a separate identity from the elements they are associated with; this should make it more efficient to implement such features as default attributes associated with all elements of a given type. Furthermore, \mbox{\hyperlink{classPoco_1_1XML_1_1Attr}{Attr}} nodes may not be immediate children of a \mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}}. However, they can be associated with \mbox{\hyperlink{classPoco_1_1XML_1_1Element}{Element}} nodes contained within a \mbox{\hyperlink{classPoco_1_1XML_1_1DocumentFragment}{Document\+Fragment}}. In short, users and implementors of the D\+OM need to be aware that \mbox{\hyperlink{classPoco_1_1XML_1_1Attr}{Attr}} nodes have some things in common with other objects inheriting the \mbox{\hyperlink{classPoco_1_1XML_1_1Node}{Node}} interface, but they also are quite distinct.

The attribute\textquotesingle{}s effective value is determined as follows\+: if this attribute has been explicitly assigned any value, that value is the attribute\textquotesingle{}s effective value; otherwise, if there is a declaration for this attribute, and that declaration includes a default value, then that default value is the attribute\textquotesingle{}s effective value; otherwise, the attribute does not exist on this element in the structure model until it has been explicitly added. Note that the node\+Value attribute on the \mbox{\hyperlink{classPoco_1_1XML_1_1Attr}{Attr}} instance can also be used to retrieve the string version of the attribute\textquotesingle{}s value(s).

In \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}}, where the value of an attribute can contain entity references, the child nodes of the \mbox{\hyperlink{classPoco_1_1XML_1_1Attr}{Attr}} node provide a representation in which entity references are not expanded. These child nodes may be either \mbox{\hyperlink{classPoco_1_1XML_1_1Text}{Text}} or \mbox{\hyperlink{classPoco_1_1XML_1_1EntityReference}{Entity\+Reference}} nodes. Because the attribute type may be unknown, there are no tokenized attribute values.

If the user changes the value of the attribute (even if it ends up having the same value as the default value) then the specified flag is automatically flipped to true. To re-\/specify the attribute as the default value from the D\+TD, the user must delete the attribute. The implementation will then make a new attribute available with specified set to false and the default value (if one exists). In summary\+: \begin{DoxyVerb}* If the attribute has an assigned value in the document then specified
  is true, and the value is the assigned value.
* If the attribute has no assigned value in the document and has a default
  value in the DTD, then specified is false, and the value is the default
  value in the DTD.
* If the attribute has no assigned value in the document and has a value

  model of the document.
* If the attribute is not associated to any element (i.e. because it
