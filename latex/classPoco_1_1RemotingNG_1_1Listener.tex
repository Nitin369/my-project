Check whether the given \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}} can be handled by this \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Listener}{Listener}} instance.

This is used by the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1ORB}{O\+RB}} to check whether a given remote service object identified by the \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}} is actually located on the same \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1ORB}{O\+RB}}.

In some cases, simply comparing the \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}} against all locally registered U\+R\+Is does not work, as \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}} may be different even if they reference the same object. For example, the U\+R\+Is \href{http://localhost/a/b/c}{\texttt{ http\+://localhost/a/b/c}} and \href{http://127.0.0.1/a/b/c}{\texttt{ http\+://127.\+0.\+0.\+1/a/b/c}} are different, but actually reference the same resource.

An implementation of this function should examine the authority part (host and port) of the \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}} and check if it is equivalent to the endpoint handled by the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Listener}{Listener}}.

A simple implementation can always return false. In this case, local objects will only be recognized if the \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}} exactly matches the \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}} created by the \mbox{\hyperlink{classPoco_1_1RemotingNG_1_1Listener}{Listener}} when the object was registered.
