\doxysubsection{Detailed Description}
Web\+Server\+Extension handles two extensions points. To register a directory in the Webserver\+: $<$extension point=\char`\"{}osp.\+web.\+server.\+directory\char`\"{} 
                   path=\char`\"{}\+U\+R\+I\char`\"{} 
                   resource=\char`\"{}dir inside bundle\char`\"{} 
                   allow\+Specialization=\char`\"{}none$\vert$owner$\vert$all\char`\"{}
                   index=\char`\"{}index.\+html\char`\"{}$>$

To register a Request\+Handler\+Factory for a given server uri/set of uris\+: $<$extension point=\char`\"{}osp.\+web.\+server.\+requesthandler\char`\"{} 
                   path=\char`\"{}\+U\+R\+I\char`\"{}
                   class=\char`\"{}\+My\+Request\+Handler\+Factory\char`\"{} 
                   library=\char`\"{}\+My\+Lib\char`\"{}$>$

Note that the extension points are automatically removed if a bundle is stopped.

The following attributes can be specified for both extension points\+:
\begin{DoxyItemize}
\item path\+: The \mbox{\hyperlink{classPoco_1_1URI}{U\+RI}} on the server to which the resource or handler is mapped.
\item description\+: User-\/readable description of resource or handler.
\item secure\+: If \char`\"{}true\char`\"{}, require a secure (H\+T\+T\+PS) connection to access the resource.
\item realm\+: Specify authentication realm (together with permission).
\item permission\+: Specify the necessary access permission for this resource. Unless a session name is specified, H\+T\+TP Basic Authentication is used to obtain the user\textquotesingle{}s user name and password, which are then checked against the specified permission using the authorization service.
\item session\+: Specify the name of the session used for session-\/based authentication instead of H\+T\+TP Basic Authentication. If specified together with a permission, the session with the specified name is obtained from the \mbox{\hyperlink{classPoco_1_1OSP_1_1Web_1_1WebSessionService}{Web\+Session\+Service}}, and the current users\textquotesingle{}s name is obtained from the session\textquotesingle{}s \char`\"{}username\char`\"{} attribute, which must be a std\+::string.
\item hidden\+: If \char`\"{}true\char`\"{}, path is not included by \mbox{\hyperlink{classPoco_1_1OSP_1_1Web_1_1WebServerDispatcher_a5e01726e3816964580964c5f01837c13}{Web\+Server\+Dispatcher\+::list\+Virtual\+Paths()}}.
\end{DoxyItemize}

The following attributes can be specified for \char`\"{}osp.\+web.\+server.\+directory\char`\"{}\+:
\begin{DoxyItemize}
\item resource\+: Specify a directory within the bundle where the H\+T\+ML, image and other files are located.
\item index\+: Specify the name of the default document (defaults to \char`\"{}index.\+html\char`\"{}).
\item cache\+: If \char`\"{}true\char`\"{}, files in this directory can be cached. Defaults to \char`\"{}true\char`\"{}. Set to \char`\"{}false\char`\"{} to disable caching.
\item allow\+Specialization\+: Using the $<$\mbox{[}allow\+Specialization\mbox{]}$>$ attribute, a bundle can specify whether other bundles can register themselves for subdirectories of the directory specified in $<$\mbox{[}path\mbox{]}$>$. The following values are supported\+: none\+: it is impossible to map resources or request handlers to subdirectories; owner\+: only this bundle can map resources or request handlers to subdirectories; all\+: any bundle can map resources or request handlers to subdirectories.
\end{DoxyItemize}

The following attributes can be specified for \char`\"{}osp.\+web.\+server.\+requesthandler\char`\"{}\+:
\begin{DoxyItemize}
\item pattern\+: A regular expression for matching requests to the handler. If specified, no path attribute should be specified. If both the pattern and path attributes are given, path is ignored. Paths containing a pattern are not included by \mbox{\hyperlink{classPoco_1_1OSP_1_1Web_1_1WebServerDispatcher_a5e01726e3816964580964c5f01837c13}{Web\+Server\+Dispatcher\+::list\+Virtual\+Paths()}}.
\item methods\+: A comma-\/separated list of H\+T\+TP request methods (\char`\"{}\+G\+E\+T\char`\"{}, \char`\"{}\+P\+O\+S\+T\char`\"{}, etc.") supported by the handler. If not specified, all methods are passed to the handler.
\item class\+: The class name of the request handler factory.
\item library\+: The name of the shared library containing the request handler factory. ~\newline
 
\end{DoxyItemize}
