\subsubsection*{template$<$class T\+Key, class T\+Value, class T\+Mutex = Fast\+Mutex, class T\+Event\+Mutex = Fast\+Mutex$>$\newline
class Poco\+::\+Unique\+Expire\+Cache$<$ T\+Key, T\+Value, T\+Mutex, T\+Event\+Mutex $>$}

An \mbox{\hyperlink{classPoco_1_1UniqueExpireCache}{Unique\+Expire\+Cache}} caches entries for a given time amount. In contrast to \mbox{\hyperlink{classPoco_1_1ExpireCache}{Expire\+Cache}} which only allows to set a per cache expiration value, it allows to define expiration per Cache\+Entry. Each T\+Value object must thus offer the following method\+:

const \mbox{\hyperlink{classPoco_1_1Timestamp}{Poco\+::\+Timestamp}}\& get\+Expiration() const;

which returns the absolute timepoint when the entry will be invalidated. Accessing an object will N\+OT update this absolute expire timepoint. You can use the \mbox{\hyperlink{classPoco_1_1ExpirationDecorator}{Poco\+::\+Expiration\+Decorator}} to add the get\+Expiration method to values that do not have a get\+Expiration function.

Be careful when using an \mbox{\hyperlink{classPoco_1_1UniqueExpireCache}{Unique\+Expire\+Cache}}. A cache is often used like cache.\+has(x) followed by cache.\+get x). Note that it could happen that the \char`\"{}has\char`\"{} call works, then the current execution thread gets descheduled, time passes, the entry gets invalid, thus leading to an empty \mbox{\hyperlink{classPoco_1_1SharedPtr}{Shared\+Ptr}} being returned when \char`\"{}get\char`\"{} is invoked.
