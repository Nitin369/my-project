\doxysubsection{Detailed Description}
Create and manage a shared memory object. \begin{DoxyVerb}A SharedMemory object has value semantics, but
is implemented using a handle/implementation idiom.
Therefore, multiple SharedMemory objects can share
a single, reference counted SharedMemoryImpl object. 
\end{DoxyVerb}

Creates or connects to a shared memory object with the given name.

For maximum portability, name should be a valid Unix filename and not contain any slashes or backslashes.

An address hint can be passed to the system, specifying the desired start address of the shared memory area. Whether the hint is actually honored is, however, up to the system. Windows platform will generally ignore the hint.

If server is set to true, the shared memory region will be unlinked by calling shm\+\_\+unlink() (on P\+O\+S\+IX platforms) when the \mbox{\hyperlink{classPoco_1_1SharedMemory}{Shared\+Memory}} object is destroyed.
