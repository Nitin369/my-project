\doxysubsection{Detailed Description}
This streambuf encodes all data written to it in hex\+Binary encoding and forwards it to a connected ostream. In hex\+Binary encoding, each binary octet is encoded as a character tuple, ~\newline
 consisting of two hexadecimal digits (\mbox{[}0-\/9a-\/f\+A-\/F\mbox{]}) representing the octet code. See also\+: \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} Schema Part 2\+: Datatypes (\href{http://www.w3.org/TR/xmlschema-2/}{\texttt{ http\+://www.\+w3.\+org/\+T\+R/xmlschema-\/2/}}), section 3.\+2.\+15.

Note\+: The characters are directly written to the ostream\textquotesingle{}s streambuf, thus bypassing the ostream. The ostream\textquotesingle{}s state is therefore not updated to match the buffer\textquotesingle{}s state.
