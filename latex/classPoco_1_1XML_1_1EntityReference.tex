\doxysubsection{Detailed Description}
\mbox{\hyperlink{classPoco_1_1XML_1_1EntityReference}{Entity\+Reference}} objects may be inserted into the structure model when an entity reference is in the source document, or when the user wishes to insert an entity reference. Note that character references and references to predefined entities are considered to be expanded by the H\+T\+ML or \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} processor so that characters are represented by their \mbox{\hyperlink{classPoco_1_1Unicode}{Unicode}} equivalent rather than by an entity reference. Moreover, the \mbox{\hyperlink{namespacePoco_1_1XML}{X\+ML}} processor may completely expand references to entities while building the structure model, instead of providing \mbox{\hyperlink{classPoco_1_1XML_1_1EntityReference}{Entity\+Reference}} objects. If it does provide such objects, then for a given \mbox{\hyperlink{classPoco_1_1XML_1_1EntityReference}{Entity\+Reference}} node, it may be that there is no \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}} node representing the referenced entity. If such an \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}} exists, then the child list of the \mbox{\hyperlink{classPoco_1_1XML_1_1EntityReference}{Entity\+Reference}} node is the same as that of the \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}} node.

As for \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}} nodes, \mbox{\hyperlink{classPoco_1_1XML_1_1EntityReference}{Entity\+Reference}} nodes and all their descendants are readonly.

The resolution of the children of the \mbox{\hyperlink{classPoco_1_1XML_1_1EntityReference}{Entity\+Reference}} (the replacement value of the referenced \mbox{\hyperlink{classPoco_1_1XML_1_1Entity}{Entity}}) may be lazily evaluated; actions by the user (such as calling the child\+Nodes method on the \mbox{\hyperlink{classPoco_1_1XML_1_1EntityReference}{Entity\+Reference}} node) are assumed to trigger the evaluation.
